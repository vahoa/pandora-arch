# 完整部署操作手册

> 本文档基于项目实际配置文件逐行核对编写，覆盖从零开始到所有服务完全启动的每一步操作。

---

## 一、端口规划总览

先明确所有服务和中间件占用的端口，避免冲突：

| 组件 | 端口 | 用途 |
|------|------|------|
| **MySQL** | 3306 | 关系数据库 |
| **Redis** | 6379 | 缓存 + 短信验证码 |
| **MinIO** | 9000 (API) / 9001 (控制台) | 对象存储 |
| **Nacos** | 8848 / 9848 | 服务注册 + 配置中心（仅微服务） |
| **Sentinel Dashboard** | 8858 | 流控监控（仅微服务，可选） |
| **pandora-auth** | 9100 | 认证授权服务 |
| **pandora-start** | 8080 | 单体模式主应用 |
| **pandora-service-user** | 8081 | 用户微服务（仅微服务模式） |
| **pandora-gateway** | 9999 | API 网关（仅微服务模式） |

## 二、中间件安装部署

### 2.1 JDK 17

```bash
# Windows：下载 Oracle JDK 17 或 Eclipse Temurin 17
# https://adoptium.net/temurin/releases/?version=17

# 验证
java -version
# 输出应包含 "17.x.x"
```

### 2.2 Maven 3.8+

```bash
# 下载：https://maven.apache.org/download.cgi
# 配置 MAVEN_HOME 和 PATH

# 验证
mvn -version
```

### 2.3 MySQL 8.x

**Windows 安装：**
```bash
# 下载 MySQL Installer：https://dev.mysql.com/downloads/installer/
# 安装时选择 "Developer Default"
# 设置 root 密码为：123456
```

**验证连接：**
```bash
mysql -h 127.0.0.1 -P 3306 -u root -p123456 -e "SELECT VERSION();"
```

**如果你有多个 MySQL 实例**，必须确认 TCP 3306 端口连接的是目标实例：
```bash
# 用 TCP 方式连接（不是 socket）
mysql -h 127.0.0.1 -P 3306 -u root -p123456
```

### 2.4 Redis 6.x+

**Windows 安装：**
```bash
# 方式一：使用 Memurai（Windows 原生 Redis 替代）
# https://www.memurai.com/

# 方式二：使用 WSL2
wsl --install
# 在 WSL2 中：
sudo apt update && sudo apt install redis-server
sudo service redis-server start

# 方式三：使用 Docker
docker run -d --name redis -p 6379:6379 redis:7-alpine
```

**验证：**
```bash
redis-cli ping
# 应返回 PONG
```

### 2.5 MinIO

**Windows 安装：**
```bash
# 下载：https://dl.min.io/server/minio/release/windows-amd64/minio.exe
# 放到一个目录下，比如 D:\minio\

# 启动（指定数据目录）
D:\minio\minio.exe server D:\minio\data --console-address ":9001"
```

**启动后：**
- API 地址：http://127.0.0.1:9000
- 控制台地址：http://127.0.0.1:9001
- 用户名/密码：minioadmin / minioadmin

**创建存储桶：**
1. 打开 http://127.0.0.1:9001
2. 登录后点击 "Create Bucket"
3. 输入桶名：`corp-web`，点击 "Create"

### 2.6 Nacos 2.2.x（仅微服务模式需要）

```bash
# 下载：https://github.com/alibaba/nacos/releases
# 选择 nacos-server-2.2.3.zip

# 解压后进入 bin 目录
# Windows 单机启动：
startup.cmd -m standalone

# Linux/Mac：
sh startup.sh -m standalone
```

**验证：**
- 打开 http://localhost:8848/nacos
- 用户名/密码：nacos / nacos

### 2.7 Sentinel Dashboard（仅微服务模式，可选）

```bash
# 下载：https://github.com/alibaba/Sentinel/releases
# 选择 sentinel-dashboard-1.8.7.jar

java -jar sentinel-dashboard-1.8.7.jar --server.port=8858
```

**验证：**
- 打开 http://localhost:8858
- 用户名/密码：sentinel / sentinel

## 三、数据库初始化

在 MySQL 中执行以下两个 SQL 脚本（**必须按顺序执行**）：

```bash
# 第一步：创建数据库和基础表
mysql -h 127.0.0.1 -P 3306 -u root -p123456 < docs/sql/schema.sql

# 第二步：创建认证扩展表（社交绑定、角色、用户角色）
mysql -h 127.0.0.1 -P 3306 -u root -p123456 < docs/sql/schema-auth.sql
```

**执行后的数据库结构：**

```
ddd_platform
├── sys_user           用户表（含 nickname、avatar 扩展字段）
├── sys_social_user    社交账号绑定表
├── sys_role           角色表（预置 ROLE_ADMIN、ROLE_USER）
└── sys_user_role      用户角色关联表
```

**验证：**
```bash
mysql -h 127.0.0.1 -P 3306 -u root -p123456 -e "USE ddd_platform; SHOW TABLES;"
```

应看到 4 张表。

## 五、单体模式 —— 启动顺序与配置

### 5.1 涉及的配置文件

单体模式只需关注 **2 个配置文件**：

| 序号 | 配置文件 | 说明 |
|------|---------|------|
| 1 | `pandora-auth/src/main/resources/application.yml` | 认证服务配置 |
| 2 | `pandora-start/src/main/resources/application.yml` | 主应用配置 |

### 5.2 需要运行的中间件

| 中间件 | 是否必须 | 说明 |
|--------|---------|------|
| MySQL 8.x | **必须** | 数据库 |
| Redis 6.x+ | **必须** | 缓存 + 短信验证码 |
| MinIO | 可选 | 文件上传功能需要 |

### 5.3 配置文件核对清单

**pandora-auth/application.yml —— 必须确认的配置项：**

```yaml
server:
  port: 9100                    # 认证服务端口

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/ddd_platform   # ← 确认 MySQL 地址
    username: root                                   # ← 确认用户名
    password: 123456                                 # ← 确认密码

  data:
    redis:
      host: localhost           # ← 确认 Redis 地址
      port: 6379                # ← 确认 Redis 端口
      database: 1               # 认证服务使用 db1

  cloud:
    nacos:
      discovery:
        enabled: false          # 单体模式下保持 false（默认值）

auth:
  sms:
    mock-enabled: true          # 开发环境：短信验证码固定为 888888
  jwt:
    secret: your-secret-key     # 生产环境必须修改为强密钥
```

**pandora-start/application.yml —— 必须确认的配置项：**

```yaml
server:
  port: 8080                    # 主应用端口

spring:
  datasource:
    dynamic:
      datasource:
        master:
          url: jdbc:mysql://localhost:3306/ddd_platform  # ← 确认
          username: root                                  # ← 确认
          password: 123456                                # ← 确认

  data:
    redis:
      host: localhost           # ← 确认 Redis 地址
      port: 6379

  security:
    oauth2:
      resourceserver:
        jwt:
          issuer-uri: http://localhost:9100   # ← 必须与 pandora-auth 端口一致

minio:
  endpoint: http://127.0.0.1:9000   # ← 确认 MinIO 地址
  access-key: minioadmin
  secret-key: minioadmin
  bucket: corp-web                   # ← 确认桶已创建
```

### 5.4 启动顺序（严格按此顺序）

```
┌─────────────────────────────────────────────────────────────┐
│  步骤1：确认中间件已启动                                       │
│    MySQL (3306) ✓   Redis (6379) ✓   MinIO (9000) 可选      │
├─────────────────────────────────────────────────────────────┤
│  步骤2：启动认证服务（必须先于主应用）                           │
│    pandora-auth → 端口 9100                                      │
├─────────────────────────────────────────────────────────────┤
│  步骤3：启动主应用                                             │
│    pandora-start → 端口 8080                                     │
└─────────────────────────────────────────────────────────────┘
```

**具体命令：**

```bash
# ====== 步骤2：启动认证服务 ======

# 方式A：Maven 启动（开发环境）
cd pandora-auth
mvn spring-boot:run

# 方式B：JAR 启动（生产环境）
java -jar pandora-auth/target/pandora-auth-1.0.0.jar

# 等待看到日志：Started AuthServerApplication in x.x seconds
```

```bash
# ====== 步骤3：启动主应用（另开一个终端） ======

# 方式A：Maven 启动
cd pandora-start
mvn spring-boot:run

# 方式B：JAR 启动
java -jar pandora-start/target/pandora-start-1.0.0.jar

# 等待看到日志：Started DddPlatformApplication in x.x seconds
```

> **为什么认证服务必须先启动？**
> `pandora-start` 配置了 `spring.security.oauth2.resourceserver.jwt.issuer-uri=http://localhost:9100`，
> 启动时会尝试连接认证服务获取 JWT 公钥。如果认证服务未启动，主应用会启动失败或接口鉴权异常。

### 5.5 单体模式验证

```bash
# 1. 认证服务健康检查
curl http://localhost:9100/actuator/health

# 2. 用户名密码登录
curl -X POST http://localhost:9100/auth/login \
  -H "Content-Type: application/json" \
  -d "{\"loginType\":\"USERNAME_PASSWORD\",\"username\":\"admin\",\"password\":\"admin123\"}"

# 3. 短信验证码（模拟模式，验证码固定 888888）
curl -X POST "http://localhost:9100/auth/sms/send?phone=13800000001"
curl -X POST http://localhost:9100/auth/login \
  -H "Content-Type: application/json" \
  -d "{\"loginType\":\"SMS_CODE\",\"phone\":\"13800000001\",\"smsCode\":\"888888\"}"

# 4. 主应用 Swagger 文档
# 浏览器打开：http://localhost:8080/swagger-ui.html

# 5. 主应用健康检查
curl http://localhost:8080/actuator/health

# 6. 使用登录获取的 token 调用业务接口
curl -H "Authorization: Bearer <步骤2获取的accessToken>" \
  http://localhost:8080/api/users/1
```

## 六、微服务模式 —— 启动顺序与配置

### 6.1 涉及的配置文件

微服务模式需要配置 **4 个配置文件** + **Nacos 配置中心**：

| 序号 | 配置文件 | 模块 | 说明 |
|------|---------|------|------|
| 1 | `pandora-auth/src/main/resources/application.yml` | 认证服务 | 需开启 Nacos |
| 2 | `pandora-gateway/src/main/resources/application.yml` | API 网关 | 路由规则 |
| 3 | `pandora-service/pandora-service-user/src/main/resources/application.yml` | 用户微服务 | 数据源 + Nacos |
| 4 | Nacos 配置中心（可选） | — | 集中管理配置 |

### 6.2 需要运行的中间件

| 中间件 | 是否必须 | 说明 |
|--------|---------|------|
| MySQL 8.x | **必须** | 所有服务共享 |
| Redis 6.x+ | **必须** | 缓存 + 短信验证码 |
| Nacos 2.2.x | **必须** | 服务注册与配置中心 |
| MinIO | 可选 | 文件上传功能 |
| Sentinel Dashboard | 可选 | 流控监控面板 |

### 6.3 各服务配置要点

**① pandora-auth（认证服务）—— 需要修改的地方：**

启动时需要设置环境变量 `NACOS_ENABLED=true`：

```bash
java -jar pandora-auth-1.0.0.jar --NACOS_ENABLED=true
# 或
set NACOS_ENABLED=true && mvn spring-boot:run
```

配置文件中的关键项：
```yaml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848      # ← Nacos 地址
        enabled: ${NACOS_ENABLED:false}  # ← 设为 true
```

**② pandora-gateway（API 网关）—— 路由规则说明：**

```yaml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848   # ← 确认 Nacos 地址

    gateway:
      routes:
        - id: pandora-service-user        # 用户服务路由
          uri: lb://pandora-service-user   # lb:// 表示通过 Nacos 负载均衡
          predicates:
            - Path=/api/users/**       # 匹配路径

        - id: pandora-auth-server          # 认证服务路由
          uri: lb://pandora-auth-server
          predicates:
            - Path=/auth/**,/oauth2/**,/.well-known/**

    sentinel:
      transport:
        dashboard: localhost:8858      # ← Sentinel 地址（可选）
```

**③ pandora-service-user（用户微服务）：**

```yaml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848   # ← 确认 Nacos 地址
      config:
        server-addr: localhost:8848
        file-extension: yml

  datasource:
    dynamic:
      datasource:
        master:
          url: jdbc:mysql://localhost:3306/ddd_platform  # ← 确认
          username: root
          password: 123456

  security:
    oauth2:
      resourceserver:
        jwt:
          issuer-uri: http://localhost:9100  # ← 确认认证服务地址
```

### 6.4 Nacos 配置中心设置（可选但推荐）

如果希望通过 Nacos 集中管理配置：

1. 打开 http://localhost:8848/nacos → 配置管理 → 配置列表
2. 点击 "+" 创建配置

**配置一：pandora-service-user.yml**
```
Data ID: pandora-service-user.yml
Group: DEFAULT_GROUP
格式: YAML
内容:
```

```yaml
spring:
  datasource:
    dynamic:
      datasource:
        master:
          url: jdbc:mysql://localhost:3306/ddd_platform?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
          username: root
          password: 123456
  data:
    redis:
      host: localhost
      port: 6379
```

**配置二：pandora-gateway.yml（可选）**
```
Data ID: pandora-gateway.yml
Group: DEFAULT_GROUP
格式: YAML
内容:
```

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: pandora-service-user
          uri: lb://pandora-service-user
          predicates:
            - Path=/api/users/**,/api/files/**,/api/ai/**
        - id: pandora-auth-server
          uri: lb://pandora-auth-server
          predicates:
            - Path=/auth/**,/oauth2/**
```

> **注意**：Nacos 中的配置会与本地 `application.yml` **合并**，Nacos 优先级更高。
> 本地 `application.yml` 中已有 `spring.config.import: optional:nacos:xxx.yml`，
> 表示从 Nacos 拉取配置，`optional` 表示 Nacos 不可用时不报错。

### 6.5 启动顺序（严格按此顺序）

```
┌─────────────────────────────────────────────────────────────────┐
│  步骤1：启动基础中间件                                            │
│    MySQL (3306) ✓  Redis (6379) ✓  Nacos (8848) ✓               │
│    MinIO (9000) 可选   Sentinel (8858) 可选                      │
├─────────────────────────────────────────────────────────────────┤
│  步骤2：启动认证服务（注册到 Nacos）                                │
│    pandora-auth → 端口 9100，--NACOS_ENABLED=true                    │
├─────────────────────────────────────────────────────────────────┤
│  步骤3：启动业务微服务                                             │
│    pandora-service-user → 端口 8081（自动注册到 Nacos）                │
├─────────────────────────────────────────────────────────────────┤
│  步骤4：启动 API 网关（最后启动，需要发现上游服务）                    │
│    pandora-gateway → 端口 9999                                       │
└─────────────────────────────────────────────────────────────────┘
```

**具体命令（每个服务单独一个终端）：**

```bash
# ====== 步骤1：确认中间件 ======
# MySQL、Redis 已运行
# 启动 Nacos：
cd nacos/bin
startup.cmd -m standalone

# ====== 步骤2：启动认证服务 ======
cd pandora-auth
mvn spring-boot:run -Dspring-boot.run.arguments="--NACOS_ENABLED=true"
# 或 JAR 方式：
java -jar pandora-auth/target/pandora-auth-1.0.0.jar --NACOS_ENABLED=true

# 等待日志：Started AuthServerApplication
# 验证：打开 Nacos 控制台 → 服务管理 → 服务列表，应看到 pandora-auth-server

# ====== 步骤3：启动用户微服务 ======
cd pandora-service/pandora-service-user
mvn spring-boot:run
# 或 JAR 方式：
java -jar pandora-service/pandora-service-user/target/pandora-service-user-1.0.0.jar

# 验证：Nacos 服务列表应看到 pandora-service-user

# ====== 步骤4：启动 API 网关 ======
cd pandora-gateway
mvn spring-boot:run
# 或 JAR 方式：
java -jar pandora-gateway/target/pandora-gateway-1.0.0.jar

# 验证：Nacos 服务列表应看到 pandora-gateway
```

> **PowerShell 特殊注意**：如果使用 `mvn spring-boot:run -D...` 参数，需要加引号：
> ```powershell
> mvn spring-boot:run "-Dspring-boot.run.arguments=--NACOS_ENABLED=true"
> ```

### 6.6 微服务模式验证

```bash
# 1. Nacos 控制台确认3个服务已注册
```

## 八、环境变量速查表

所有敏感配置均支持通过环境变量覆盖，无需修改配置文件：

| 环境变量 | 对应配置 | 默认值 | 说明 |
|---------|---------|--------|------|
| `NACOS_ENABLED` | 认证服务 Nacos 注册 | false | 微服务模式设为 true |
| `AUTH_JWT_SECRET` | JWT 签名密钥 | 内置默认值 | 生产环境必须修改 |
| `SMS_MOCK_ENABLED` | 短信模拟模式 | true | 生产设为 false |
| `AUTH_BASE_URL` | 社交登录回调基地址 | http://localhost:9100 | 生产设为公网地址 |
| `OPENAI_API_KEY` | Spring AI 密钥 | 空 | 需要 AI 功能时配置 |
| `OPENAI_BASE_URL` | OpenAI 基地址 | https://api.openai.com | 可指向代理地址 |
| `OPENAI_MODEL` | 使用的 AI 模型 | gpt-4o-mini | — |

## 九、常见启动失败排查

### 9.1 "Access denied for user 'root'@'localhost'"

MySQL 密码不对。确认 `application.yml` 中密码与实际一致，并用 TCP 方式验证：
```bash
mysql -h 127.0.0.1 -P 3306 -u root -p123456
```

### 9.2 "Unknown database 'ddd_platform'"

数据库未初始化。执行：
```bash
mysql -h 127.0.0.1 -P 3306 -u root -p123456 < docs/sql/schema.sql
mysql -h 127.0.0.1 -P 3306 -u root -p123456 < docs/sql/schema-auth.sql
```

### 9.3 "Connection refused: localhost:6379"

Redis 未启动。确认 Redis 正在运行：
```bash
redis-cli ping
```

### 9.4 "Connection refused: localhost:9100"（主应用启动失败）

认证服务未启动或端口不对。**必须先启动 pandora-auth**，再启动 pandora-start。

### 9.5 "Port 9000 already in use"（pandora-auth 或其他）

MinIO 默认占用 9000 端口。认证服务已改为 9100，不应冲突。如果仍有冲突检查端口占用：
```bash
netstat -ano | findstr :9000
```

### 9.6 Nacos 注册失败

1. 确认 Nacos 已启动：http://localhost:8848/nacos
2. 确认 `NACOS_ENABLED=true` 已设置
3. 确认 `spring.cloud.nacos.discovery.server-addr` 地址正确

### 9.7 网关路由不生效（微服务模式）

1. 确认上游服务（pandora-auth-server、pandora-service-user）已在 Nacos 注册
2. 检查网关 `application.yml` 中路由的 `Path` 和 `uri` 是否正确
3. 网关必须**最后启动**（上游服务先注册）

## 十、两种模式对比决策

| 维度 | 单体模式 | 微服务模式 |
|------|---------|-----------|
| 启动的服务数 | 2 个（auth + start） | 4 个（auth + user + gateway + Nacos） |
| 必须的中间件 | MySQL + Redis | MySQL + Redis + Nacos |
| 配置复杂度 | 低（2个yml） | 中（4个yml + Nacos可选） |
| 适用场景 | 开发/测试/小型项目 | 生产/大型项目/团队协作 |
| 扩缩容 | 整体扩容 | 按服务独立扩容 |
| 访问入口 | http://localhost:8080 | http://localhost:9999（网关） |

> **建议**：开发阶段使用单体模式（快速启动），生产环境使用微服务模式（弹性扩展）。
