# 完整部署操作手册

> 本文档基于项目实际配置文件逐行核对编写，覆盖从零开始到所有服务完全启动的每一步操作。
> 基线：JDK 25 + Spring Boot 4.0.5 + Spring Cloud Alibaba 2025.1.0.0 + MyBatis-Flex 1.11.6 + Redisson 4.3.1

---

## 一、端口规划总览

先明确所有服务和中间件占用的端口，避免冲突：

| 组件 | 业务端口 | 运维端口 | 用途 |
|------|---------|---------|------|
| **MySQL 9** | 3306 | — | 关系数据库（`mysql-connector-j 9.6.0`） |
| **Redis 7** | 6379 | — | 缓存 / Token 黑白名单 / 验证码（仅 Redisson 客户端） |
| **MinIO** | 9000 / 9001 | — | 对象存储（API / Console） |
| **Nacos 集群** | 8140 / 8550 / 8960 | — | 注册 + 配置中心（默认 `192.168.50.168` 3 节点） |
| **Sentinel Dashboard** | 8858 | — | 流控监控（可选） |
| **pandora-auth-server** | 9100 | 19100 | 认证授权服务（Spring AS 7.0.4） |
| **pandora-arch（pandora-start）** | 8080 | 18080 | 单体模式主应用 |
| **pandora-service-user** | 8081 | 18082 | 用户微服务（仅微服务） |
| **pandora-gateway** | 8888 | 18888 | API 网关（Spring Cloud Gateway Server WebFlux，仅微服务） |

## 二、中间件安装部署

### 2.1 JDK 25

```bash
# Windows：下载 Oracle JDK 25 或 Eclipse Temurin 25
# https://adoptium.net/temurin/releases/?version=25

# 验证
java -version
# 输出应包含 "25"
```

> 本项目强制使用 JDK 25（LTS）；Lombok 锁定到 `1.18.44`，更早版本在 JDK 25 下注解处理会失败。
> 构建插件已为 Lombok 打开 `jdk.compiler` 模块（`--add-opens`），消除 `sun.misc.Unsafe` 警告。

### 2.2 Maven 3.9+

```bash
# 下载：https://maven.apache.org/download.cgi
# 配置 MAVEN_HOME 和 PATH

# 验证
mvn -version
```

### 2.3 MySQL 8.x / 9.x

**Windows 安装：**
```bash
# 下载 MySQL Installer：https://dev.mysql.com/downloads/installer/
# 推荐 MySQL 9.x（本项目驱动版本 mysql-connector-j 9.6.0 完全兼容）
# 安装时选择 "Developer Default"
# 设置 root 密码为：123456
```

**验证连接：**
```bash
mysql -h 127.0.0.1 -P 3306 -u root -p123456 -e "SELECT VERSION();"
```

**如果你有多个 MySQL 实例**，必须确认 TCP 3306 端口连接的是目标实例：
```bash
# 用 TCP 方式连接（不是 socket）
mysql -h 127.0.0.1 -P 3306 -u root -p123456
```

### 2.4 Redis 7.x

> 本项目只使用 **Redisson 4.3.1** 作为客户端（`redisson-spring-boot-starter` + `redisson-spring-data-40`），**严禁使用 Lettuce/Jedis**。

**Windows 安装：**
```bash
# 方式一：使用 Memurai（Windows 原生 Redis 替代）
# https://www.memurai.com/

# 方式二：使用 WSL2
wsl --install
# 在 WSL2 中：
sudo apt update && sudo apt install redis-server
sudo service redis-server start

# 方式三：使用 Docker
docker run -d --name redis -p 6379:6379 redis:7-alpine
```

**验证：**
```bash
redis-cli ping
# 应返回 PONG
```

## 三、数据库初始化

在 MySQL 中执行以下两个 SQL 脚本（**必须按顺序执行**）：

```bash
# 第一步：创建数据库和基础表
mysql -h 127.0.0.1 -P 3306 -u root -p123456 < docs/sql/schema.sql

# 第二步：创建认证扩展表（社交绑定、角色、用户角色）
mysql -h 127.0.0.1 -P 3306 -u root -p123456 < docs/sql/schema-auth.sql
```

**执行后的数据库结构：**

```
ddd_platform
├── sys_user           用户表（含 nickname、avatar 扩展字段）
├── sys_social_user    社交账号绑定表
├── sys_role           角色表（预置 ROLE_ADMIN、ROLE_USER）
└── sys_user_role      用户角色关联表
```

**验证：**
```bash
mysql -h 127.0.0.1 -P 3306 -u root -p123456 -e "USE ddd_platform; SHOW TABLES;"
```

应看到 4 张表。

## 五、单体模式 —— 启动顺序与配置

### 5.1 涉及的配置文件

单体模式只需关注 **2 个配置文件**：

| 序号 | 配置文件 | 说明 |
|------|---------|------|
| 1 | `pandora-auth/src/main/resources/application.yml` | 认证服务配置 |
| 2 | `pandora-start/src/main/resources/application.yml` | 主应用配置 |

### 5.2 需要运行的中间件

| 中间件 | 是否必须 | 说明 |
|--------|---------|------|
| MySQL 8.x / 9.x | 必须 | 数据库 |
| Redis 7.x | 必须 | 缓存 + 短信验证码（Redisson 客户端） |
| MinIO | 可选 | 文件上传功能需要 |

### 5.4 启动顺序（严格按此顺序）

```
┌─────────────────────────────────────────────────────────────┐
│  步骤1：确认中间件已启动                                       │
│    MySQL (3306) ✓   Redis (6379) ✓   MinIO (9000) 可选      │
├─────────────────────────────────────────────────────────────┤
│  步骤2：启动认证服务（必须先于主应用）                           │
│    pandora-auth → 端口 9100                                      │
├─────────────────────────────────────────────────────────────┤
│  步骤3：启动主应用                                             │
│    pandora-start → 端口 8080                                     │
└─────────────────────────────────────────────────────────────┘
```

**具体命令：**

```bash
# ====== 步骤2：启动认证服务 ======

# 方式A：Maven 启动（开发环境）
cd pandora-auth
mvn spring-boot:run

# 方式B：JAR 启动（生产环境）
java -jar pandora-auth/target/pandora-auth-1.0.0.jar

# 等待看到日志：Started AuthServerApplication in x.x seconds
```

```bash
# ====== 步骤3：启动主应用（另开一个终端） ======

# 方式A：Maven 启动
cd pandora-start
mvn spring-boot:run

# 方式B：JAR 启动
java -jar pandora-start/target/pandora-start-1.0.0.jar

# 等待看到日志：Started PandoraApplication in x.x seconds
```

> **为什么认证服务必须先启动？**
> 项目采用 HMAC 共享密钥的 JWT 签名方案，`pandora-start` 作为资源服务器需要加载与 `pandora-auth` 相同的 `auth.jwt.secret`。
> 建议先启动认证服务确保密钥/初始化账号就绪，再启动主应用；若配置了 `issuer-uri` 的 OIDC 模式，则主应用启动时会尝试连接认证服务获取公钥。

## 六、微服务模式 —— 启动顺序与配置

### 6.1 涉及的配置文件

微服务模式需要配置 **4 个配置文件** + **Nacos 配置中心**：

| 序号 | 配置文件 | 模块 | 说明 |
|------|---------|------|------|
| 1 | `pandora-auth/src/main/resources/application.yml` | 认证服务 | 需开启 Nacos |
| 2 | `pandora-gateway/src/main/resources/application.yml` | API 网关 | 路由规则 |
| 3 | `pandora-service/pandora-service-user/src/main/resources/application.yml` | 用户微服务 | 数据源 + Nacos |
| 4 | Nacos 配置中心（可选） | — | 集中管理配置 |

### 6.2 需要运行的中间件

| 中间件 | 是否必须 | 说明 |
|--------|---------|------|
| MySQL 8.x / 9.x | 必须 | 所有服务共享 |
| Redis 7.x | 必须 | 缓存 + 短信验证码（Redisson） |
| Nacos 2.x（默认 3 节点集群） | 必须 | 服务注册与配置中心 |
| MinIO | 可选 | 文件上传功能 |
| Sentinel Dashboard | 可选 | 流控监控面板 |
| Kafka 3.8.x（KRaft） | 可选 | `pandora.kafka.enabled=true` 才启用 |
| MongoDB 5/6/7 | 可选 | `pandora.mongodb.enabled=true` 才启用 |

### 6.3 各服务配置要点

**① pandora-auth（认证服务）—— 需要修改的地方：**

启动时需要设置环境变量 `NACOS_ENABLED=true`：

```bash
java -jar pandora-auth-1.0.0.jar --NACOS_ENABLED=true
# 或
set NACOS_ENABLED=true && mvn spring-boot:run
```

配置文件中的关键项：
```yaml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848      # ← Nacos 地址
        enabled: ${NACOS_ENABLED:false}  # ← 设为 true
```

**② pandora-gateway（API 网关）—— 路由规则说明：**

> 本项目网关使用新 artifactId：`spring-cloud-starter-gateway-server-webflux`，
> 属性前缀从旧版 `spring.cloud.gateway.*` 改为 **`spring.cloud.gateway.server.webflux.*`**。

```yaml
server:
  port: 8888
management:
  server:
    port: 18888

spring:
  application:
    name: pandora-gateway
  cloud:
    nacos:
      discovery:
        server-addr: ${NACOS_SERVER_ADDR:192.168.50.168:8140,192.168.50.168:8550,192.168.50.168:8960}
        group: ${spring.profiles.active}
      config:
        server-addr: ${NACOS_SERVER_ADDR:192.168.50.168:8140,192.168.50.168:8550,192.168.50.168:8960}
        group: ${spring.profiles.active}
        file-extension: yml

    gateway:
      server:
        webflux:
          discovery:
            locator:
              enabled: true
              lower-case-service-id: true
          routes:
            - id: pandora-service-user
              uri: lb://pandora-service-user
              predicates:
                - Path=/api/users/**
            - id: pandora-auth-server
              uri: lb://pandora-auth-server
              predicates:
                - Path=/auth/**,/oauth2/**,/.well-known/**

    sentinel:
      transport:
        dashboard: localhost:8858
      datasource:
        ds1:
          nacos:
            server-addr: ${spring.cloud.nacos.discovery.server-addr}
            group-id: ${spring.profiles.active}
            data-id: sentinel-${spring.application.name}-flow.json
            rule-type: flow
```

### 6.4 Nacos 配置中心设置（可选但推荐）

如果希望通过 Nacos 集中管理配置：

1. 打开 http://localhost:8848/nacos → 配置管理 → 配置列表
2. 点击 "+" 创建配置

**Group 约定：** `dev` / `test` / `prod`（对应 `spring.profiles.active`），**严禁使用 `DEFAULT_GROUP`**。
**Data ID 命名：** `{spring.application.name}-{profile}.yml`。

**配置一：pandora-service-user-dev.yml**
```
Data ID: pandora-service-user-dev.yml
Group: dev
格式: YAML
内容:
```

```yaml
spring:
  datasource:
    dynamic:
      datasource:
        master:
          url: jdbc:mysql://localhost:3306/ddd_platform?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
          username: root
          password: 123456
  data:
    redis:
      host: localhost
      port: 6379
```

**配置二：pandora-gateway-dev.yml**
```
Data ID: pandora-gateway-dev.yml
Group: dev
格式: YAML
内容:
```

```yaml
spring:
  cloud:
    gateway:
      server:
        webflux:
          routes:
            - id: pandora-service-user
              uri: lb://pandora-service-user
              predicates:
                - Path=/api/users/**,/api/files/**,/api/ai/**
            - id: pandora-auth-server
              uri: lb://pandora-auth-server
              predicates:
                - Path=/auth/**,/oauth2/**
```

> **注意**：Nacos 中的配置会与本地 `application.yml` **合并**，Nacos 优先级更高。
> 本地 `application.yml` 中已有 `spring.config.import: optional:nacos:xxx.yml`，
> 表示从 Nacos 拉取配置，`optional` 表示 Nacos 不可用时不报错。

### 6.5 启动顺序（严格按此顺序）

```
┌─────────────────────────────────────────────────────────────────┐
│  步骤1：启动基础中间件                                            │
│    MySQL (3306) ✓  Redis (6379) ✓  Nacos (8848) ✓               │
│    MinIO (9000) 可选   Sentinel (8858) 可选                      │
├─────────────────────────────────────────────────────────────────┤
│  步骤2：启动认证服务（注册到 Nacos）                                │
│    pandora-auth → 端口 9100，--NACOS_ENABLED=true                    │
├─────────────────────────────────────────────────────────────────┤
│  步骤3：启动业务微服务                                             │
│    pandora-service-user → 端口 8081（自动注册到 Nacos）                │
├─────────────────────────────────────────────────────────────────┤
│  步骤4：启动 API 网关（最后启动，需要发现上游服务）                    │
│    pandora-gateway → 端口 8888（运维 18888）                          │
└─────────────────────────────────────────────────────────────────┘
```

**具体命令（每个服务单独一个终端）：**

```bash
# ====== 步骤1：确认中间件 ======
# MySQL、Redis 已运行
# 启动 Nacos：
cd nacos/bin
startup.cmd -m standalone

# ====== 步骤2：启动认证服务 ======
cd pandora-auth
mvn spring-boot:run -Dspring-boot.run.arguments="--NACOS_ENABLED=true"
# 或 JAR 方式：
java -jar pandora-auth/target/pandora-auth-1.0.0.jar --NACOS_ENABLED=true

# 等待日志：Started AuthServerApplication
# 验证：打开 Nacos 控制台 → 服务管理 → 服务列表，应看到 pandora-auth-server

# ====== 步骤3：启动用户微服务 ======
cd pandora-service/pandora-service-user
mvn spring-boot:run
# 或 JAR 方式：
java -jar pandora-service/pandora-service-user/target/pandora-service-user-1.0.0.jar

# 验证：Nacos 服务列表应看到 pandora-service-user

# ====== 步骤4：启动 API 网关 ======
cd pandora-gateway
mvn spring-boot:run
# 或 JAR 方式：
java -jar pandora-gateway/target/pandora-gateway-1.0.0.jar

# 验证：Nacos 服务列表应看到 pandora-gateway
```

> **PowerShell 特殊注意**：如果使用 `mvn spring-boot:run -D...` 参数，需要加引号：
> ```powershell
> mvn spring-boot:run "-Dspring-boot.run.arguments=--NACOS_ENABLED=true"
> ```

## 九、常见启动失败排查

### 9.1 "Access denied for user 'root'@'localhost'"

MySQL 密码不对。确认 `application.yml` 中密码与实际一致，并用 TCP 方式验证：
```bash
mysql -h 127.0.0.1 -P 3306 -u root -p123456
```

### 9.2 "Unknown database 'ddd_platform'"

数据库未初始化。执行：
```bash
mysql -h 127.0.0.1 -P 3306 -u root -p123456 < docs/sql/schema.sql
mysql -h 127.0.0.1 -P 3306 -u root -p123456 < docs/sql/schema-auth.sql
```

### 9.3 "Connection refused: localhost:6379"

Redis 未启动。确认 Redis 正在运行：
```bash
redis-cli ping
```

### 9.4 "Connection refused: localhost:9100"（主应用启动失败）

认证服务未启动或端口不对。**必须先启动 pandora-auth**，再启动 pandora-start。

### 9.5 "Port 9000 already in use"（pandora-auth 或其他）

MinIO 默认占用 9000 端口。认证服务已改为 9100，不应冲突。如果仍有冲突检查端口占用：
```bash
netstat -ano | findstr :9000
```

### 9.6 Nacos 注册失败

1. 确认 Nacos 已启动：`NACOS_SERVER_ADDR` 指向的地址可连
2. 确认 `NACOS_ENABLED=true` 已设置（Auth 服务）
3. 确认 Group 与 `spring.profiles.active` 一致（不是 DEFAULT_GROUP）

### 9.7 网关路由不生效（微服务模式）

1. 确认上游服务（pandora-auth-server、pandora-service-user）已在 Nacos 注册
2. 必须使用新属性前缀 `spring.cloud.gateway.server.webflux.*`
3. 检查路由的 `Path` 和 `uri=lb://<nacos 服务名>` 是否正确
4. 网关必须**最后启动**（上游服务先注册）

### 9.8 Lombok 注解处理失败（JDK 25）

- 升级到 **Lombok 1.18.44**（父 POM 已锁定）
- 构建插件已配置 `--add-opens=jdk.compiler/com.sun.tools.javac.processing=ALL-UNNAMED`
- IDE 需安装 Lombok 插件（IntelliJ IDEA 2025.1+ 自带）

### 9.9 找不到 `commons-logging`

Spring Framework 7 移除 `spring-jcl`，父 POM 已统一兜底引入 `commons-logging`。如果业务模块出现 `ClassNotFoundException: org.apache.commons.logging.LogFactory`，检查是否有模块排除了该依赖。

### 9.10 Redis 客户端相关报错（Lettuce/Jedis）

本项目**只允许 Redisson**（`redisson-spring-boot-starter` + `redisson-spring-data-40`）。若业务模块出现 `io.lettuce.core.*` 相关类，说明引入了 Lettuce，需要排除：

```xml
<exclusion>
    <groupId>io.lettuce</groupId>
    <artifactId>lettuce-core</artifactId>
</exclusion>
```

## 十、两种模式对比决策

| 维度 | 单体模式 | 微服务模式 |
|------|---------|-----------|
| 启动的服务数 | 2 个（auth + start） | 4 个（auth + user + gateway + Nacos） |
| 必须的中间件 | MySQL + Redis | MySQL + Redis + Nacos |
| 配置复杂度 | 低（2 个 yml） | 中（4 个 yml + Nacos 远程配置） |
| 适用场景 | 开发 / 测试 / 小型项目 | 生产 / 大型项目 / 团队协作 |
| 扩缩容 | 整体扩容 | 按服务独立扩容 |
| 访问入口 | http://localhost:8080 | http://localhost:8888（网关） |
| 服务间调用 | 方法调用 | OpenFeign（推荐） / RestClient + NamingService / Dubbo |

> **建议**：开发阶段使用单体模式（快速启动），生产环境使用微服务模式（弹性扩展）。
