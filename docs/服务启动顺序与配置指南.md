# 服务启动顺序与配置指南

> 本文档说明单体模式和微服务模式下的完整配置和启动步骤。
> 基线：JDK 25 + Spring Boot 4.0.5 + Spring Cloud Alibaba 2025.1.0.0 + MyBatis-Flex 1.11.6 + Redisson 4.3.1

---

## 一、中间件依赖

在启动任何服务之前，必须先确保以下中间件正常运行：

| 中间件 | 地址 | 用途 | 必须？ |
|--------|------|------|-------|
| MySQL 8.x / 9.x | localhost:3306 | 主数据库（驱动 `mysql-connector-j 9.6.0`） | 必须 |
| Redis 7.x | localhost:6379 | 缓存/Token黑名单/验证码/权限缓存（仅 Redisson 客户端） | 必须 |
| MinIO | 127.0.0.1:9000 / 9001 | 对象存储 | 可选（不用文件功能） |
| Nacos 2.x（默认 3 节点集群） | `192.168.50.168:8140,8550,8960` | 注册 + 配置中心 | 仅微服务模式 |
| Sentinel Dashboard | localhost:8858 | 流控控制台 | 微服务模式可选 |
| Kafka 3.8.x（KRaft） | localhost:9092 | 消息 | 可选（`pandora.kafka.enabled`） |
| MongoDB 5/6/7 | localhost:27017 | 文档库 | 可选（`pandora.mongodb.enabled`） |

### 1.1 MySQL 初始化

```sql
CREATE DATABASE IF NOT EXISTS ddd_platform DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

USE ddd_platform;
SOURCE docs/sql/schema.sql;         -- 基础表
SOURCE docs/sql/schema-auth.sql;    -- OAuth2 授权表
SOURCE docs/sql/schema-rbac.sql;    -- RBAC 权限表
SOURCE docs/sql/schema-member.sql;  -- C 端会员表
```

**初始账号：**

- B 端管理员：`admin / admin123`
- C 端测试会员：手机号 `13900000001`，验证码 `888888`（Mock 模式）

### 1.2 Redis 配置

所有服务必须使用同一 Redis database（默认 database **1**）。

```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
      database: 1
      password: your-password   # 如有
```

> 项目只使用 **Redisson 4.3.1** 作为 Redis 客户端：`redisson-spring-boot-starter` + `redisson-spring-data-40`。严禁使用 Lettuce / Jedis。

### 1.3 MinIO 配置

```bash
docker run -d --name minio \
  -p 9000:9000 -p 9001:9001 \
  -e "MINIO_ROOT_USER=minioadmin" \
  -e "MINIO_ROOT_PASSWORD=minioadmin" \
  minio/minio server /data --console-address ":9001"
```

控制台 http://127.0.0.1:9001，创建桶 `corp-web`。

## 二、单体模式启动

### 2.1 启动顺序

```
第 1 步：中间件（MySQL + Redis）
第 2 步：pandora-auth   ← 端口 9100 / 运维 19100，独立部署
第 3 步：pandora-start  ← 端口 8080 / 运维 18080，聚合 DDD 四层 + Infra
```

### 2.2 详细步骤

**第 1 步：确认中间件**

```bash
mysql -u root -p123456 -P 3306 -e "SELECT 1"
redis-cli -p 6379 PING                       # 应返回 PONG
```

**第 2 步：启动 Auth 服务**

```bash
mvn -pl pandora-auth spring-boot:run \
  -Dspring-boot.run.arguments="--spring.profiles.active=dev"
```

启动成功日志：`Started AuthServerApplication in X seconds`。

**原理：** Auth 服务是独立 Spring Boot 应用，基于 **Spring Authorization Server 7.0.4**：

- 接收登录 → 验证身份 → 签发 JWT（HMAC）
- RefreshToken 存入 Redis 白名单；登出时 AccessToken jti 加入 Redis 黑名单
- 支持代次（tgen）机制强制踢人
- 启动时可通过密码迁移 Runner 自动升级加密算法

**第 3 步：启动单体服务**

```bash
mvn -pl pandora-start spring-boot:run \
  -Dspring-boot.run.arguments="--spring.profiles.active=dev"
```

启动成功日志：`Started PandoraApplication in X seconds`。

**原理：** `pandora-start` 通过 Maven 依赖聚合 `pandora-api` + `pandora-infrastructure`（传递 `pandora-application` + `pandora-domain` + `pandora-common`）。Spring Boot 启动时：

1. 自动配置多数据源（基于 **dynamic-datasource-spring-boot4-starter 4.5.0**）
2. 初始化 **MyBatis-Flex 1.11.6** + Spring Data JPA（双 ORM 共存）
3. 注册 Redisson 客户端与 `RedisTemplate`（由 `redisson-spring-data-40` 支持）
4. 注册多条 `SecurityFilterChain`（公共/B端/C端/兜底）
5. 每条 Chain 注入 `LoginUserFilter`（jti 黑名单 + tgen 代次校验）
6. 注册 AOP 切面 `PermissionAspect` 处理权限注解
7. 注册 MyBatis 标准拦截器 `DataPermissionInterceptor` 处理数据权限

**第 4 步：验证**

```powershell
# B 端登录
$body = @{ loginType="USERNAME_PASSWORD"; username="admin"; password="admin123" } | ConvertTo-Json
Invoke-RestMethod -Uri "http://localhost:9100/auth/admin/login" -Method Post -Body $body -ContentType "application/json"

# Swagger UI
# http://localhost:8080/swagger-ui/index.html
# 健康检查
# http://localhost:18080/actuator/health
```

### 2.3 关键配置文件

| 文件 | 位置 | 说明 |
|------|------|-----|
| application.yml | pandora-auth/src/main/resources/ | Auth 服务配置（JWT、SMS、JustAuth、Redis） |
| application.yml | pandora-start/src/main/resources/ | 单体服务配置（多数据源、Redis、MinIO、MyBatis-Flex） |
| application-dev.yml | pandora-start/src/main/resources/ | 开发环境覆盖 |

**核心配置项：**

```yaml
# Auth 服务 (pandora-auth)
server:
  port: 9100
management:
  server:
    port: 19100

auth:
  jwt:
    secret: pandora-arch-jwt-secret-key-must-be-at-least-256-bits-long-2026
    expiration: 7200            # AccessToken TTL
    refresh-expiration: 604800  # RefreshToken TTL
  sms:
    mock-enabled: true          # Mock，验证码固定 888888

# 单体服务 (pandora-start)
server:
  port: 8080
management:
  server:
    port: 18080

spring:
  datasource:
    dynamic:
      primary: master
      datasource:
        master:
          url: jdbc:mysql://localhost:3306/ddd_platform?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
        slave:
          url: jdbc:mysql://localhost:3306/ddd_platform?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true

  data:
    redis:
      host: localhost
      port: 6379
      database: 1

auth.jwt.secret: 同 Auth 服务    # 必须与 Auth 一致
```

## 三、微服务模式启动

### 3.1 额外中间件

```
Nacos 集群:   192.168.50.168:8140,8550,8960    ← 注册 + 配置 + Sentinel 规则源
Sentinel Dashboard: localhost:8858（可选）
```

Nacos 单机启动示例：

```bash
docker run -d --name nacos \
  -e MODE=standalone \
  -p 8848:8848 -p 9848:9848 \
  nacos/nacos-server:v2.4.0

# 控制台 http://localhost:8848/nacos  (nacos / nacos)
```

本地若连不到内网集群，可设置 `NACOS_SERVER_ADDR=127.0.0.1:8848` 使用单机 Nacos。

### 3.2 Nacos 配置（Group = 环境 Profile）

**Group 约定：** `dev` / `test` / `prod`（**不要使用 `DEFAULT_GROUP`**）

**Data ID 规则：** `{spring.application.name}-{profile}.yml`

| 服务 | Data ID | Group |
|------|---------|-------|
| Auth | `pandora-auth-server-dev.yml` | `dev` |
| User 微服务 | `pandora-service-user-dev.yml` | `dev` |
| 网关 | `pandora-gateway-dev.yml` | `dev` |

**`pandora-gateway-dev.yml` 示例（新属性前缀）：**

```yaml
spring:
  cloud:
    gateway:
      server:
        webflux:
          discovery:
            locator:
              enabled: true
              lower-case-service-id: true
          routes:
            - id: pandora-auth
              uri: lb://pandora-auth-server
              predicates:
                - Path=/auth/**
            - id: pandora-service-user
              uri: lb://pandora-service-user
              predicates:
                - Path=/api/users/**
```

### 3.3 启动顺序

```
第 1 步：中间件（MySQL + Redis + Nacos）
第 2 步：pandora-auth :9100      ← NACOS_ENABLED=true，注册服务名 pandora-auth-server
第 3 步：pandora-service-user :8081    ← 注册服务名 pandora-service-user
第 4 步：pandora-gateway :8888   ← 从 Nacos 发现服务
```

**启动命令：**

```bash
# Auth
mvn -pl pandora-auth spring-boot:run \
  -Dspring-boot.run.arguments="--NACOS_ENABLED=true --spring.profiles.active=dev"

# User 微服务
mvn -pl pandora-service/pandora-service-user spring-boot:run \
  -Dspring-boot.run.arguments="--spring.profiles.active=dev"

# Gateway
mvn -pl pandora-gateway spring-boot:run \
  -Dspring-boot.run.arguments="--spring.profiles.active=dev"
```

### 3.4 微服务架构流程

```
客户端 → Gateway :8888 → Nacos 发现 → pandora-auth-server :9100 / pandora-service-user :8081 ...
   │
   ├── /auth/**  → lb://pandora-auth-server
   ├── /api/**   → lb://pandora-service-user
   │
   └── AuthGlobalFilter
          ├── 白名单直接放行（/auth/**、/actuator/**、Swagger）
          ├── Bearer Token 校验 + jti 黑名单 + tgen 代次
          └── 解析后放入请求头透传下游
```

### 3.5 微服务与单体的差异

| 配置项 | 单体 | 微服务 |
|--------|------|--------|
| 入口端口 | 8080 (`pandora-start`) | 8888 (`pandora-gateway`) |
| Nacos | 不使用 | 必须开启 |
| 配置来源 | 本地 `application.yml` | Nacos Config |
| 网关 | 无 | Spring Cloud Gateway Server WebFlux |
| 服务间调用 | 方法调用 | **OpenFeign**（首选）/ RestClient + NamingService / Dubbo |
| 负载均衡 | 无 | Spring Cloud LoadBalancer（仅网关）；业务侧自行基于 `NamingService` |
| 熔断限流 | 无 | Sentinel（`sentinel-datasource-nacos`） |

## 五、常见启动问题

| 问题 | 原因 | 解决 |
|------|-----|-----|
| Port already in use | 上次未关闭 | `netstat -ano | findstr :8080` → `taskkill /PID <pid> /F` |
| Cannot connect to MySQL | 未启动 / 端口不对 | `mysql -P 3306` |
| Cannot connect to Redis | 未启动 | `redis-cli PING` → `PONG` |
| JWT 验证失败 | Auth 和资源服务 secret 不一致 | 统一 `AUTH_JWT_SECRET` |
| Token 黑名单不生效 | Redis database 不一致 | 所有服务统一 `database: 1` |
| Nacos 注册不上 | 集群地址不通 / Group 不一致 | 确认 `NACOS_SERVER_ADDR` + Group = profile |
| 网关属性警告 | 用了旧前缀 | 必须 `spring.cloud.gateway.server.webflux.*` |
| Lombok 注解处理器失败 | Lombok < 1.18.44 | 升级到 `1.18.44`，构建插件已配 `--add-opens` |
| ClassNotFoundException: ChatClient | 未引入 Spring AI 依赖 | 正常，`AiAutoConfiguration` 条件装配会跳过 |
