# 服务启动顺序与配置指南

> 本文档说明单体模式和微服务模式下的完整配置和启动步骤。

---

## 一、中间件依赖

在启动任何服务之前，必须先确保以下中间件正常运行：

| 中间件 | 地址 | 用途 | 必须？ |
|--------|------|------|--------|
| MySQL 8.x | localhost:3306 | 主数据库 | ✅ 必须 |
| Redis 7.x | localhost:6379 | 缓存/Token黑名单/验证码/权限缓存 | ✅ 必须 |
| MinIO | 127.0.0.1:9000 | 文件存储 | ⚠️ 可选（不用文件功能可不启动） |
| Nacos 2.x | localhost:8848 | 注册中心+配置中心 | ⚠️ 仅微服务模式需要 |
| Sentinel Dashboard | localhost:8858 | 限流熔断控制台 | ⚠️ 仅微服务模式可选 |

### 1.1 MySQL 配置

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS ddd_platform DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

-- 按顺序执行初始化脚本
USE ddd_platform;
SOURCE docs/sql/schema.sql;       -- 基础表
SOURCE docs/sql/schema-auth.sql;  -- OAuth2 授权表
SOURCE docs/sql/schema-rbac.sql;  -- RBAC 权限表（部门/角色/菜单/关联表 + 初始数据）
SOURCE docs/sql/schema-member.sql; -- C端会员表
```

**初始账号：**
- B端管理员：`admin / admin123`
- C端测试会员：手机号 `13900000001`，验证码 `888888`（Mock 模式）

### 1.2 Redis 配置

默认配置即可。注意：**所有服务必须使用同一个 Redis database（默认 database 1）**。

如果 Redis 有密码，需要在 `application.yml` 中配置：
```yaml
spring.data.redis:
  host: localhost
  port: 6379
  database: 1
  password: your-password  # 如果有密码
```

### 1.3 MinIO 配置

```bash
# Docker 快速启动 MinIO
docker run -d --name minio \
  -p 9000:9000 -p 9001:9001 \
  -e "MINIO_ROOT_USER=minioadmin" \
  -e "MINIO_ROOT_PASSWORD=minioadmin" \
  minio/minio server /data --console-address ":9001"
```

启动后在 http://127.0.0.1:9001 创建名为 `corp-web` 的存储桶。

## 二、单体模式启动

### 2.1 启动顺序

```
第1步：中间件（MySQL + Redis）
第2步：Auth 服务（端口 9100）        ← 认证授权，独立部署
第3步：Start 单体服务（端口 8080）    ← 所有业务模块
```

### 2.2 详细步骤

**第1步：确认中间件**

```bash
# 确认 MySQL
mysql -u root -p123456 -P 3306 -e "SELECT 1"

# 确认 Redis
redis-cli -p 6379 PING   # 应返回 PONG
```

**第2步：启动 Auth 服务**

```bash
cd 项目根目录
mvn spring-boot:run -pl pandora-auth -DskipTests
```

启动成功标志：`Started AuthServerApplication in X seconds`，端口 9100。

**原理：** Auth 服务是独立的 Spring Boot 应用（`AuthServerApplication`），负责：
- 接收登录请求 → 验证身份 → 签发 JWT（HMAC 签名）
- 将 RefreshToken 存入 Redis 白名单
- 提供登出接口（AccessToken 加入 Redis 黑名单）
- 启动时通过 `PasswordMigrationRunner` 自动加密明文密码

**第3步：启动单体服务**

```bash
mvn spring-boot:run -pl pandora-start -DskipTests
```

启动成功标志：`Started PandoraApplication in X seconds`，端口 8080。

**原理：** Start 模块通过 Maven 依赖聚合了 `pandora-api`、`pandora-infrastructure`、`pandora-application`、`pandora-domain` 四个 DDD 层。Spring Boot 启动时：
1. 自动配置多数据源（master/slave）
2. 初始化 MyBatis-Plus + JPA 双 ORM
3. 注册 4 条 SecurityFilterChain（公共/B端/C端/兜底）
4. 在每条 FilterChain 中注入 `LoginUserFilter`（含 Token 黑名单校验）
5. 注册 AOP 切面（`PermissionAspect`）处理权限注解
6. 注册 MyBatis-Plus 拦截器（`DataPermissionInterceptor`）处理数据权限

**第4步：验证**

```powershell
# B端登录
$body = @{ loginType="USERNAME_PASSWORD"; username="admin"; password="admin123" } | ConvertTo-Json
Invoke-RestMethod -Uri "http://localhost:9100/auth/admin/login" -Method Post -Body $body -ContentType "application/json"

# 访问 Swagger UI
# 浏览器打开 http://localhost:8080/swagger-ui.html
```

### 2.3 关键配置文件

| 文件 | 位置 | 说明 |
|------|------|------|
| application.yml | pandora-auth/src/main/resources/ | Auth 服务配置（JWT 密钥、SMS、社交登录、Redis database=1） |
| application.yml | pandora-start/src/main/resources/ | 单体服务配置（多数据源、Redis database=1、MinIO、Swagger） |
| application-dev.yml | pandora-start/src/main/resources/ | 开发环境覆盖（debug 日志） |

**核心配置项：**

```yaml
# Auth 服务 (pandora-auth)
auth:
  jwt:
    secret: pandora-arch-jwt-secret-key-must-be-at-least-256-bits-long-2024  # JWT 签名密钥
    expiration: 7200       # AccessToken 有效期（秒）
    refresh-expiration: 604800  # RefreshToken 有效期（秒）
  sms:
    mock-enabled: true     # Mock 模式，验证码固定为 888888

# 单体服务 (pandora-start)
spring.datasource.dynamic:
  primary: master          # 默认数据源
  datasource:
    master:
      url: jdbc:mysql://localhost:3306/ddd_platform...
    slave:
      url: jdbc:mysql://localhost:3306/ddd_platform...  # 读写分离时配置从库

auth.jwt.secret: 同上      # 必须与 Auth 服务的密钥完全一致！
spring.data.redis.database: 1  # 必须与 Auth 服务的 Redis database 一致！
```

## 三、微服务模式启动

### 3.1 额外中间件

```
Nacos:    localhost:8848    ← 服务注册发现 + 配置中心
Sentinel: localhost:8858    ← 限流熔断控制台（可选）
```

**Nacos 安装：**

```bash
# Docker 启动 Nacos（单机模式）
docker run -d --name nacos \
  -e MODE=standalone \
  -p 8848:8848 -p 9848:9848 \
  nacos/nacos-server:v2.3.0

# 访问控制台: http://localhost:8848/nacos
# 默认账号: nacos / nacos
```

### 3.2 Nacos 配置

在 Nacos 配置中心创建以下配置：

**Data ID: `pandora-auth-server.yml`**

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/ddd_platform...
    username: root
    password: 123456
  data:
    redis:
      host: localhost
      port: 6379
      database: 1
auth:
  jwt:
    secret: pandora-arch-jwt-secret-key-must-be-at-least-256-bits-long-2024
```

**Data ID: `pandora-service-user.yml`**

```yaml
spring:
  datasource:
    dynamic:
      datasource:
        master:
          url: jdbc:mysql://localhost:3306/ddd_platform...
  data:
    redis:
      host: localhost
      port: 6379
      database: 1
auth:
  jwt:
    secret: pandora-arch-jwt-secret-key-must-be-at-least-256-bits-long-2024
```

**Data ID: `pandora-gateway.yml`**

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: auth-service
          uri: lb://pandora-auth-server
          predicates:
            - Path=/auth/**
        - id: user-service
          uri: lb://pandora-service-user
          predicates:
            - Path=/api/**
```

### 3.3 启动顺序

```
第1步：中间件（MySQL + Redis + Nacos）
第2步：Auth 服务（9100）                    ← 注册到 Nacos
第3步：业务服务（pandora-service-user 等）       ← 注册到 Nacos
第4步：Gateway 网关（8888）                  ← 从 Nacos 发现服务
```

**启动命令：**

```bash
# Auth 服务（启用 Nacos）
NACOS_ENABLED=true mvn spring-boot:run -pl pandora-auth -DskipTests

# 用户服务
mvn spring-boot:run -pl pandora-service/pandora-service-user -DskipTests

# 网关
mvn spring-boot:run -pl pandora-gateway -DskipTests
```

### 3.4 微服务架构流程

```
客户端 → Gateway (8888) → Nacos 服务发现 → Auth 服务 (9100) / 用户服务 / ...
           │
           ├── /auth/**  → 转发到 pandora-auth-server
           ├── /api/**   → 转发到 pandora-service-user
           │
           └── AuthGlobalFilter
                  ├── 白名单路径直接放行（/auth/login 等）
                  ├── 其他路径验证 JWT 有效性
                  └── 将用户信息放入请求头传递给下游
```

### 3.5 微服务与单体的区别

| 配置项 | 单体 | 微服务 |
|--------|------|--------|
| 入口端口 | 8080 (pandora-start) | 8888 (pandora-gateway) |
| Nacos | 关闭 (`NACOS_ENABLED=false`) | 开启 |
| 配置来源 | 本地 application.yml | Nacos Config |
| 服务调用 | 直接方法调用 | OpenFeign RPC |
| 负载均衡 | 无 | Spring Cloud LoadBalancer |
| 熔断限流 | 无 | Sentinel |

## 四、配置项速查表

### 4.1 必须保持一致的配置

以下配置在所有服务中**必须完全一致**，否则会导致认证失败：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `auth.jwt.secret` | JWT HMAC 签名密钥 | `pandora-arch-jwt-secret-key-must-be-at-least-256-bits-long-2024` |
| `spring.data.redis.database` | Redis 数据库编号 | `1` |
| `spring.data.redis.host` | Redis 地址 | `localhost` |
| `spring.data.redis.port` | Redis 端口 | `6379` |

### 4.2 环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `AUTH_JWT_SECRET` | JWT 密钥（覆盖配置文件） | — |
| `NACOS_ENABLED` | 是否启用 Nacos | `false` |
| `SMS_MOCK_ENABLED` | 短信是否 Mock | `true` |
| `WECHAT_MINI_APP_ID` | 微信小程序 AppID | — |
| `WECHAT_MINI_APP_SECRET` | 微信小程序 AppSecret | — |
| 社交登录系列 | `WECHAT_APP_CLIENT_ID`、`QQ_CLIENT_ID` 等 | — |

## 五、常见启动问题

| 问题 | 原因 | 解决 |
|------|------|------|
| Port already in use | 上次服务未关闭 | `Kill-Port 9100` / `Kill-Port 8080` |
| Cannot connect to MySQL | MySQL 未启动或端口不对 | 确认 `mysql -P 3306` 可连接 |
| Cannot connect to Redis | Redis 未启动 | 确认 `redis-cli PING` 返回 PONG |
| JWT 验证失败 | Auth 和 Start 的 jwt.secret 不一致 | 统一密钥配置 |
| Token 黑名单不生效 | Redis database 不一致 | 所有服务统一 `database: 1` |
| ClassNotFoundException: ChatClient | Spring AI 可选依赖不存在 | 正常行为，AiAutoConfiguration 会跳过 |
