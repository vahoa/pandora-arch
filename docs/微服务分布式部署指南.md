# 微服务分布式部署指南

## 项目背景

本项目基于 **Spring Boot 3.2.3 + Spring Cloud Alibaba 2023 + DDD（领域驱动设计）** 的架构底座，支持微服务分布式部署。

微服务模式涉及以下服务：

| 服务 | 端口 | 说明 |
|------|------|------|
| **pandora-auth** | 9100 | 认证授权服务（支持 9 种登录方式） |
| **pandora-gateway** | 9000 | API 网关 |
| **pandora-service-user** | 8081 | 用户微服务 |
| **Nacos** | 8848 | 服务注册与配置中心 |
| **MySQL** | 3306 | 关系型数据库 |
| **Redis** | 6379 | 缓存与会话存储 |
| **MinIO** | 9000 | 对象存储（与网关端口冲突时需调整） |

## 2. 环境要求

| 组件 | 版本要求 | 默认地址 |
|------|----------|----------|
| **JDK** | 17+ | - |
| **Maven** | 3.8+ | - |
| **MySQL** | 8.x | localhost:3306 |
| **Redis** | 6.x+ | localhost:6379 |
| **MinIO** | 最新 | 127.0.0.1:9000 |
| **Nacos** | 2.2.x | localhost:8848 |
| **Sentinel Dashboard** | 可选 | localhost:8858 |

> **注意**：MinIO 默认端口为 9000，与网关端口冲突时，需将 MinIO 调整为其他端口（如 9001），或调整网关端口。

## 5. 各服务配置说明

### 5.1 pandora-auth（认证服务）

**端口**：9100

**关键配置**：

| 配置项 | 说明 |
|--------|------|
| `NACOS_ENABLED` | 必须设为 `true` 启用服务注册 |
| `spring.cloud.nacos.discovery.server-addr` | Nacos 地址 localhost:8848 |
| `spring.data.redis` | Redis 配置（会话、验证码存储） |
| `auth.jwt.secret` | JWT 密钥，生产环境务必修改 |
| `auth.sms.mock-enabled` | 短信 mock 模式，生产环境设为 false |
| `auth.social.*` | 微信、QQ 等社交登录配置 |

**多渠道认证**：支持用户名密码、手机验证码、微信小程序、微信 APP、微信扫码、微信公众号、企业微信、QQ 等 9 种登录方式。

### 5.2 pandora-gateway（API 网关）

**端口**：9000（与 MinIO 冲突时需调整 MinIO 端口）

**关键配置**：

| 配置项 | 说明 |
|--------|------|
| `spring.cloud.nacos.discovery` | Nacos 服务发现 |
| `spring.cloud.gateway.routes` | 路由规则 |
| `spring.cloud.gateway.discovery.locator.enabled` | 启用基于服务名的动态路由 |

**路由规则**：

- `/api/users/**` → `lb://pandora-service-user`（用户服务）
- `/auth/**`、`/oauth2/**`、`/.well-known/**` → `lb://pandora-auth-server`（认证服务）

**认证过滤器**：`AuthGlobalFilter` 拦截非白名单请求，校验 `Authorization: Bearer <token>` 头。白名单路径直接放行：`/auth/`、`/oauth2/`、`/login`、`/.well-known/`、`/actuator/`。

### 5.3 pandora-service-user（用户微服务）

**端口**：8081

**关键配置**：

| 配置项 | 说明 |
|--------|------|
| `spring.cloud.nacos.discovery` | Nacos 注册 |
| `spring.cloud.nacos.config` | Nacos 配置中心 |
| `spring.cloud.sentinel` | Sentinel 流控 Dashboard |
| `spring.datasource.dynamic` | 动态多数据源（master/slave） |
| `spring.security.oauth2.resourceserver.jwt.issuer-uri` | JWT 验签地址（认证服务） |
| `minio.*` | MinIO 对象存储配置 |

**OpenFeign**：通过 `@FeignClient` 声明式调用其他微服务，`FeignConfig` 自动传递 `Authorization` 头。

## 7. 启动顺序

**必须按以下顺序启动**，否则可能出现服务注册失败或路由不可用：

| 顺序 | 服务 | 说明 |
|------|------|------|
| 1 | Nacos | 服务注册中心必须先启动 |
| 2 | MySQL / Redis / MinIO | 基础设施 |
| 3 | pandora-auth | 认证服务（其他服务依赖 JWT 验签） |
| 4 | pandora-service-user | 用户微服务 |
| 5 | pandora-gateway | API 网关（最后启动，依赖服务发现） |

**各服务启动命令**：

```bash
# 1. 认证服务（必须启用 Nacos）
java -jar pandora-auth/target/pandora-auth-*.jar --NACOS_ENABLED=true

# 2. 用户微服务（微服务模式默认已启用 Nacos）
java -jar pandora-service/pandora-service-user/target/pandora-service-user-*.jar --NACOS_ENABLED=true

# 3. API 网关
java -jar pandora-gateway/target/pandora-gateway-*.jar
```

使用 Maven 启动时：

```bash
cd pandora-auth
mvn spring-boot:run -Dspring-boot.run.arguments="--NACOS_ENABLED=true"

cd pandora-service/pandora-service-user
mvn spring-boot:run -Dspring-boot.run.arguments="--NACOS_ENABLED=true"

cd pandora-gateway
mvn spring-boot:run
```

## 8. 服务验证

| 验证项 | 地址 | 说明 |
|--------|------|------|
| Nacos 控制台 | http://localhost:8848/nacos | 账号 nacos/nacos，查看服务列表 |
| 网关健康检查 | http://localhost:9000/actuator/health | 返回 `{"status":"UP"}` |
| 通过网关访问用户服务 | http://localhost:9000/api/users/1 | 需携带 Bearer Token |
| 通过网关认证 | POST http://localhost:9000/auth/login | 用户名密码登录 |
| OAuth2 获取令牌 | POST http://localhost:9000/oauth2/token | 客户端凭证模式 |

**获取令牌示例**：

```bash
curl -X POST http://localhost:9000/oauth2/token \
  -u "pandora-service-client:pandora-service-secret" \
  -d "grant_type=client_credentials&scope=read write"
```

**通过网关访问用户 API**：

```bash
curl -H "Authorization: Bearer <access_token>" \
  http://localhost:9000/api/users/1
```

## 9. Sentinel 流控配置

### 9.1 启动 Sentinel Dashboard

```bash
java -jar sentinel-dashboard-1.8.7.jar --server.port=8858
```

控制台：http://localhost:8858（默认账号 sentinel/sentinel）

### 9.2 配置类型

| 类型 | 说明 | 应用场景 |
|------|------|----------|
| **接口级别限流** | QPS/并发数限制 | 保护核心接口 |
| **熔断降级** | 异常比例/慢调用熔断 | 防止雪崩 |
| **热点参数限流** | 针对特定参数限流 | 如按 userId 限流 |

### 9.3 规则配置

在 Sentinel Dashboard 中为 `pandora-service-user` 等服务配置流控规则，规则会推送到各微服务。触发限流时返回统一格式：

```json
{
  "code": 429,
  "message": "请求过于频繁，请稍后重试",
  "data": null,
  "timestamp": 1709712000000
}
```

## 10. 服务间通信

### 10.1 OpenFeign 声明式调用

**定义 Feign 客户端**：

```java
@FeignClient(name = "pandora-service-user", path = "/api/users")
public interface UserFeignClient {

    @GetMapping("/{id}")
    Result<UserDTO> getUser(@PathVariable("id") Long id);
}
```

**使用**：

```java
@Service
public class OrderApplicationService {

    private final UserFeignClient userFeignClient;

    public OrderApplicationService(UserFeignClient userFeignClient) {
        this.userFeignClient = userFeignClient;
    }

    public void createOrder(CreateOrderCommand command) {
        Result<UserDTO> userResult = userFeignClient.getUser(command.getUserId());
        // ...
    }
}
```

**JWT 传递**：`pandora-common-cloud` 的 `FeignConfig` 已配置全局拦截器，自动将当前请求的 `Authorization` 头传递到 Feign 调用中。

### 10.2 负载均衡策略

默认使用 Spring Cloud LoadBalancer，基于 `lb://服务名` 进行客户端负载均衡，策略为轮询（RoundRobin）。

## 11. 配置中心使用

### 11.1 Nacos 配置管理

在 Nacos 控制台「配置管理 → 配置列表」中创建配置：

- **Data ID**：`pandora-service-user.yml`（与 `spring.application.name` 一致）
- **Group**：`DEFAULT_GROUP`
- **配置格式**：YAML

应用通过以下配置自动拉取：

```yaml
spring:
  config:
    import:
      - optional:nacos:${spring.application.name}.yml
```

### 11.2 动态配置刷新

使用 `@RefreshScope` 或 `@ConfigurationProperties` 的配置支持动态刷新，无需重启服务。

### 11.3 多环境配置

| 环境 | Data ID 示例 | 说明 |
|------|--------------|------|
| dev | `pandora-service-user-dev.yml` | 开发环境 |
| test | `pandora-service-user-test.yml` | 测试环境 |
| prod | `pandora-service-user-prod.yml` | 生产环境 |

通过 `spring.profiles.active` 或 Nacos 的 `spring.cloud.nacos.config.namespace` 区分环境。

## 12. Docker 部署（可选）

### 12.1 docker-compose.yml 示例

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: pandora-mysql
    ports:
      - "3306:3306"
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: ddd_platform
    volumes:
      - mysql_data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: pandora-redis
    ports:
      - "6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  nacos:
    image: nacos/nacos-server:v2.2.3
    container_name: pandora-nacos
    ports:
      - "8848:8848"
      - "9848:9848"
    environment:
      MODE: standalone
      SPRING_DATASOURCE_PLATFORM: ""
      JVM_XMS: 256m
      JVM_XMX: 256m
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8848/nacos/"]
      interval: 10s
      timeout: 5s
      retries: 10

  minio:
    image: minio/minio:latest
    container_name: pandora-minio
    ports:
      - "9001:9000"  # 避免与网关 9000 冲突
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin

      - minio_data:/data

      context: .
      dockerfile: Dockerfile.auth
    container_name: pandora-auth

      - "9100:9100"
    environment:
      NACOS_ENABLED: "true"
      SPRING_CLOUD_NACOS_DISCOVERY_SERVER_ADDR: nacos:8848
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/ddd_platform?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai

      SPRING_DATASOURCE_PASSWORD: 123456
      SPRING_DATA_REDIS_HOST: redis
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy
      nacos:
        condition: service_healthy

  pandora-service-user:
    build:
      context: .
      dockerfile: Dockerfile.user
    container_name: pandora-service-user
    ports:
      - "8081:8081"
    environment:
      NACOS_ENABLED: "true"
      SPRING_CLOUD_NACOS_DISCOVERY_SERVER_ADDR: nacos:8848
      SPRING_DATASOURCE_DYNAMIC_DATASOURCE_MASTER_URL: jdbc:mysql://mysql:3306/ddd_platform?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
      SPRING_DATASOURCE_DYNAMIC_DATASOURCE_MASTER_USERNAME: root
      SPRING_DATASOURCE_DYNAMIC_DATASOURCE_MASTER_PASSWORD: 123456
      SPRING_DATA_REDIS_HOST: redis
      SPRING_SECURITY_OAUTH2_RESOURCESERVER_JWT_ISSUER_URI: http://pandora-auth:9100
      MINIO_ENDPOINT: http://minio:9000

        condition: service_healthy
      redis:
        condition: service_healthy

        condition: service_healthy

    build:
      context: .
      dockerfile: Dockerfile.gateway
    container_name: pandora-gateway

      - "9000:9000"

      SPRING_CLOUD_NACOS_DISCOVERY_SERVER_ADDR: nacos:8848

        condition: service_healthy
```

## 13. 生产环境部署清单

| 项目 | 检查项 |
|------|--------|
| **Nacos** | 集群模式（至少 3 节点），持久化到 MySQL |
| **Redis** | 哨兵或集群模式，禁用危险命令 |
| **MySQL** | 主从复制，定期备份 |
| **JWT Secret** | 修改 `AUTH_JWT_SECRET` 为强密钥（≥256 位） |
| **短信** | 关闭 `SMS_MOCK_ENABLED`，配置真实短信服务商 |
| **MinIO** | 分布式模式，生产存储桶权限隔离 |
| **网关** | Nginx + Gateway 双层，SSL 终结、限流、WAF |
| **日志** | 集中式日志（ELK/Loki），链路追踪（SkyWalking） |
| **监控** | Prometheus + Grafana，告警规则配置 |

## 14. 常见问题排查

### 14.1 服务注册不上 Nacos

| 可能原因 | 排查步骤 |
|----------|----------|
| NACOS_ENABLED 未启用 | 启动时加 `--NACOS_ENABLED=true` |
| Nacos 未启动或不可达 | 检查 `localhost:8848` 是否可访问 |
| 网络/防火墙 | 检查 8848、9848 端口是否开放 |
| 命名空间错误 | 确认 `namespace: public` 或与 Nacos 控制台一致 |

### 14.2 网关路由不生效

| 可能原因 | 排查步骤 |
|----------|----------|
| 下游服务未注册 | 在 Nacos 控制台确认服务已注册 |
| 路由路径不匹配 | 检查 `Path` 谓词与请求路径是否一致 |
| 网关启动顺序 | 确保先启动 pandora-auth、pandora-service-user，再启动网关 |
| StripPrefix 配置 | 根据是否需要保留前缀调整 `StripPrefix` |

### 14.3 Sentinel 规则不生效

| 可能原因 | 排查步骤 |
|----------|----------|
| Dashboard 未连接 | 检查 `spring.cloud.sentinel.transport.dashboard` 地址 |
| 端口 8719 冲突 | 多实例时每个实例需不同 `port` |
| 规则未推送 | 在 Dashboard 配置后需「推送规则」到对应服务 |
| 依赖未引入 | 确认 `pandora-common-cloud` 已引入 Sentinel 依赖 |

### 14.4 跨服务调用超时

| 可能原因 | 排查步骤 |
|----------|----------|
| Feign 超时过短 | 配置 `feign.client.config.default.connectTimeout`、`readTimeout` |
| 网络延迟 | 检查服务间网络、DNS 解析 |
| 下游服务阻塞 | 检查下游服务日志、数据库慢查询 |
| 熔断触发 | 查看 Sentinel 熔断规则是否过于严格 |
