# 微服务分布式部署指南

## 项目背景

本项目基于 **JDK 25 + Spring Boot 4.0.5 + Spring Cloud 2025.1.0 + Spring Cloud Alibaba 2025.1.0.0 + DDD** 构建，支持微服务分布式部署。

微服务模式涉及以下服务：

| 服务（Nacos 服务名） | 业务端口 | 运维端口 | 说明 |
|------|------|------|------|
| `pandora-auth-server` | **9100** | 19100 | 认证授权服务（Spring Authorization Server 7.0.4） |
| `pandora-gateway` | **8888** | 18888 | API 网关（Spring Cloud Gateway Server WebFlux） |
| `pandora-service-user` | **8081** | 18082 | 用户微服务 |
| **Nacos 集群** | 8140 / 8550 / 8960 | — | 服务注册 + 配置中心（默认 3 节点：`192.168.50.168:8140,8550,8960`） |
| **MySQL 9** | 3306 | — | 关系型数据库 |
| **Redis 7** | 6379 | — | 缓存/会话（仅 Redisson 客户端） |
| **MinIO** | 9000 / 9001 | — | 对象存储（API/Console） |
| **Sentinel Dashboard** | 8858（可选） | — | 流控控制台 |

> **网关端口**：本项目已把网关业务端口设为 **8888**，运维 **18888**，避免与 MinIO 9000 冲突。

## 5. 各服务配置说明

### 5.1 pandora-auth（认证服务）

**端口**：业务 9100 / 运维 19100；**Nacos 服务名**：`pandora-auth-server`。

| 配置项 | 说明 |
|--------|-----|
| `NACOS_ENABLED` | 设为 `true` 启用服务注册（微服务模式必开） |
| `spring.cloud.nacos.discovery.server-addr` | Nacos 地址（默认 3 节点集群） |
| `spring.cloud.nacos.discovery.group` | `${spring.profiles.active}` |
| `spring.data.redis` | Redis 配置（验证码 / 黑名单 / 白名单） |
| `auth.jwt.secret` | JWT HMAC 密钥，生产必须覆盖 |
| `auth.sms.mock-enabled` | 短信模拟开关 |
| `justauth.type.*` | 微信 / QQ / 企业微信 社交登录 |

**多渠道认证**：用户名密码、短信验证码、微信小程序、微信 APP、微信扫码、微信公众号、企业微信、QQ 等。

### 5.2 pandora-gateway（API 网关）

**端口**：业务 8888 / 运维 18888；**Nacos 服务名**：`pandora-gateway`。

| 配置项 | 说明 |
|--------|-----|
| `spring.cloud.nacos.discovery` | Nacos 服务发现 |
| `spring.cloud.nacos.config` | Nacos 配置中心（拉取路由规则） |
| `spring.cloud.gateway.server.webflux.routes` | 路由规则（新属性前缀） |
| `spring.cloud.gateway.server.webflux.discovery.locator.enabled` | 基于服务名动态路由 |
| `spring.cloud.sentinel.*` | Sentinel Dashboard 对接 |

**artifactId 说明**：`pandora-gateway` 使用 `spring-cloud-starter-gateway-server-webflux`（新版），属性前缀为 `spring.cloud.gateway.server.webflux.*`。

**路由规则（示例）**：

```yaml
spring:
  cloud:
    gateway:
      server:
        webflux:
          discovery:
            locator:
              enabled: true
              lower-case-service-id: true
          routes:
            - id: pandora-auth
              uri: lb://pandora-auth-server
              predicates:
                - Path=/auth/**
            - id: pandora-service-user
              uri: lb://pandora-service-user
              predicates:
                - Path=/api/users/**
```

**认证过滤器**：`AuthGlobalFilter` 拦截非白名单请求，校验 `Authorization: Bearer <token>`。白名单：`/auth/**`、`/oauth2/**`、`/.well-known/**`、`/actuator/**`、`/v3/api-docs/**`、`/swagger-ui/**`。

### 5.3 pandora-service-user（用户微服务）

**端口**：业务 8081 / 运维 18082；**Nacos 服务名**：`pandora-service-user`。

| 配置项 | 说明 |
|--------|-----|
| `spring.cloud.nacos.discovery` | Nacos 注册 |
| `spring.cloud.nacos.config` | Nacos 配置中心 |
| `spring.cloud.sentinel.*` | Sentinel 流控对接 |
| `spring.datasource.dynamic` | 动态多数据源（master/slave） |
| `mybatis-flex.*` | MyBatis-Flex 配置 |
| `spring.data.redis` | Redis + Redisson |
| `spring.security.oauth2.resourceserver.jwt.*` | JWT 资源服务器（HMAC 共享密钥） |
| `pandora.minio.*` | MinIO 配置 |

**远程调用**：基于 Spring Cloud Alibaba 2025.1.0.0，项目**原生支持 OpenFeign**（`spring-cloud-starter-openfeign` + LoadBalancer + Sentinel 熔断），为首选方式；也可选 `RestClient` + Nacos `NamingService` 或 Dubbo，三者可在不同微服务并存。

## 7. 启动顺序

**必须按以下顺序启动**：

| 顺序 | 服务 | 说明 |
|------|------|------|
| 1 | Nacos 集群 | 服务注册中心 |
| 2 | MySQL / Redis / MinIO | 基础设施（按需 Kafka / MongoDB） |
| 3 | pandora-auth | 认证服务（注册到 Nacos） |
| 4 | pandora-service-user | 用户微服务（及其他业务服务） |
| 5 | pandora-gateway | 网关最后启动（依赖服务发现） |

**各服务启动命令**：

```bash
# 推荐 JVM 参数（JDK 25）
JAVA_OPTS="-XX:+UseZGC -XX:+ZGenerational \
  -Dspring.threads.virtual.enabled=true \
  --add-opens=jdk.compiler/com.sun.tools.javac.processing=ALL-UNNAMED"

# 1. 认证服务
java $JAVA_OPTS -jar pandora-auth/target/pandora-auth-1.0.0.jar \
  --NACOS_ENABLED=true --spring.profiles.active=dev

# 2. 用户微服务
java $JAVA_OPTS -jar pandora-service/pandora-service-user/target/pandora-service-user-1.0.0.jar \
  --spring.profiles.active=dev

# 3. API 网关
java $JAVA_OPTS -jar pandora-gateway/target/pandora-gateway-1.0.0.jar \
  --spring.profiles.active=dev
```

Maven 启动：

```bash
mvn -pl pandora-auth spring-boot:run \
  -Dspring-boot.run.arguments="--NACOS_ENABLED=true --spring.profiles.active=dev"
mvn -pl pandora-service/pandora-service-user spring-boot:run \
  -Dspring-boot.run.arguments="--spring.profiles.active=dev"
mvn -pl pandora-gateway spring-boot:run \
  -Dspring-boot.run.arguments="--spring.profiles.active=dev"
```

## 10. 服务间通信

项目基于 Spring Cloud Alibaba 2025.1.0.0，服务间调用按使用便利性推荐顺序：**OpenFeign（首选） → RestClient + Nacos NamingService → Dubbo**，三者可在不同微服务混用。

### 10.1 OpenFeign（首选）

```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-openfeign</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-loadbalancer</artifactId>
</dependency>
<!-- 可选：Sentinel 熔断降级 -->
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-sentinel</artifactId>
</dependency>
```

```java
@SpringBootApplication
@EnableFeignClients(basePackages = "cn.pandora.**.client")
public class OrderServiceApplication { }

@FeignClient(name = "pandora-service-user", path = "/api/users", fallback = UserClientFallback.class)
public interface UserClient {
    @GetMapping("/{id}")
    Result<UserDTO> getById(@PathVariable("id") Long id);
}
```

```yaml
feign:
  sentinel:
    enabled: true
  client:
    config:
      default:
        connect-timeout: 3000
        read-timeout: 5000
```

`pandora-common-cloud` 提供默认 `RequestInterceptor`，自动透传 `Authorization` 头到被调方。

### 10.2 RestClient + Nacos NamingService（可选）

```java
@Component
@RequiredArgsConstructor
public class UserServiceClient {

    private final NacosServiceManager serviceManager;
    private final RestClient restClient = RestClient.create();

    public UserDTO getUser(Long id, String authToken) {
        List<Instance> instances = serviceManager
            .getNamingService()
            .selectInstances("pandora-service-user", true);
        if (instances.isEmpty()) {
            throw new BusinessException(ResultCode.SERVICE_UNAVAILABLE);
        }
        // 可自行加负载均衡：随机 / RoundRobin
        Instance inst = instances.get(ThreadLocalRandom.current().nextInt(instances.size()));
        String url = "http://" + inst.getIp() + ":" + inst.getPort() + "/api/users/" + id;
        return restClient.get().uri(url)
            .header(HttpHeaders.AUTHORIZATION, authToken)
            .retrieve().body(UserDTO.class);
    }
}
```

### 10.3 Dubbo（可选）

需引入 Dubbo Spring Boot Starter，通过 `@DubboService` / `@DubboReference` 声明服务与调用方，Nacos 作为注册中心。

> 三种方式可并存：对外部系统或跨语言调用走 OpenFeign / RestClient；对内部高性能场景走 Dubbo。

## 14. 常见问题排查

### 14.1 服务注册不上 Nacos

| 可能原因 | 排查步骤 |
|----------|---------|
| `NACOS_ENABLED` 未启用（Auth） | 启动参数加 `--NACOS_ENABLED=true` |
| Nacos 不可达 | 检查 `NACOS_SERVER_ADDR` 端口可访问 |
| Group 不一致 | Group 必须与 `spring.profiles.active` 一致 |

### 14.2 网关路由不生效

| 可能原因 | 排查步骤 |
|----------|---------|
| 下游服务未注册 | 在 Nacos 控制台查看服务列表 |
| 旧属性前缀 | 必须使用 `spring.cloud.gateway.server.webflux.*`（新版） |
| 路径谓词错误 | 检查 `Path=/xxx/**` 与请求路径是否匹配 |

### 14.3 Sentinel 规则不生效

| 可能原因 | 排查步骤 |
|----------|---------|
| Dashboard 未连接 | 检查 `spring.cloud.sentinel.transport.dashboard` |
| 规则未持久化 | 使用 `sentinel-datasource-nacos` + 在 Dashboard「推送规则」 |
| 端口 8719 冲突 | 多实例需分配不同 `spring.cloud.sentinel.transport.port` |

### 14.4 跨服务调用异常

| 可能原因 | 排查步骤 |
|----------|---------|
| OpenFeign 调用超时 | 确认引入 `spring-cloud-starter-loadbalancer`；调大 `feign.client.config.default.{connect,read}-timeout`；Sentinel 熔断规则检查 |
| `Authorization` 未透传 | RestClient 调用时手动带上当前请求的 Authorization 头 |
| 下游服务超时 | 调整 RestClient 超时（`HttpClient.connectTimeout` / `Duration`） |
