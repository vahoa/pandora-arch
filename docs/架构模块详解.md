# pandora-arch — 架构模块详解

> 基线：JDK 25 + Spring Boot 4.0.5 + Spring Cloud Alibaba 2025.1.0.0
> 更新日期：2026-04

本文件介绍 `pandora-arch` 各 Maven 模块的职责、依赖关系和关键实现。

---

## 0. 模块总览

```
pandora-arch                             父 POM（统一依赖版本管理）
│
├── 基础层 ───────────────────────
├── pandora-common                       公共基础模块
├── pandora-common-cloud                 微服务公共模块（Nacos + Sentinel）
│
├── DDD 四层 ─────────────────────
├── pandora-domain                       领域层
├── pandora-application                  应用层
├── pandora-infrastructure               基础设施层
├── pandora-api                          接口层
│
├── 认证授权 ─────────────────────
├── pandora-auth                         OAuth2 授权服务器
│
├── 单体启动 ─────────────────────
├── pandora-start                        单体部署入口
│
└── 微服务启动 ───────────────────
    ├── pandora-gateway                  API 网关
    └── pandora-service                  微服务父 POM
        └── pandora-service-user         用户微服务
```

## 1. pandora-common（公共基础模块）

**职责：** 为所有模块提供最基础、零依赖的公共能力。

**包结构：**

```
cn.pandora.common
├── api                     统一响应
│   ├── Result<T>           统一响应体
│   ├── ResultCode          错误码枚举
│   ├── PageResult<T>       分页响应
│   └── PageQuery           分页查询基类
├── exception               异常体系
│   ├── BusinessException   业务异常
│   └── SystemException     系统异常
├── entity                  基础实体
│   └── BaseEntity          基础实体字段 (id, createTime, updateTime 等)
├── enums                   枚举
│   ├── UserType            用户类型 (B_USER=1, C_USER=2)
│   └── DataScopeType       数据权限范围枚举
├── security                安全上下文
│   ├── LoginUser           登录用户信息（B/C 端统一封装）
│   ├── LoginUserHolder     ThreadLocal 持有者
│   └── TokenConstants      Token 常量（Header、claim 名称等）
├── annotation              注解
│   ├── @RequiresPermission 权限校验
│   ├── @RequiresUserType   用户类型校验
│   └── @DataPermission     数据权限注解
└── util                    工具类
    ├── JsonUtils / DateUtils
    ├── SecurityUtils / ServletUtils
    ├── IdGenerator (雪花算法 / UUID)
    ├── BeanCopyUtils / AssertUtils
```

**依赖：** 仅依赖 JDK / Jackson / Hutool，**零框架依赖**。

**约束：** 不允许引用 Spring Security / MyBatis / Redisson 等任何框架实现。

---

## 2. pandora-common-cloud（微服务公共模块）

**职责：** 聚合微服务通用 Starter，让业务微服务一行依赖即可获得 Nacos + Sentinel 能力。

**核心依赖：**

```xml
<dependencies>
    <dependency>
        <groupId>com.alibaba.cloud</groupId>
        <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
    </dependency>
    <dependency>
        <groupId>com.alibaba.cloud</groupId>
        <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
    </dependency>
    <dependency>
        <groupId>com.alibaba.cloud</groupId>
        <artifactId>spring-cloud-starter-alibaba-sentinel</artifactId>
    </dependency>
    <dependency>
        <groupId>com.alibaba.csp</groupId>
        <artifactId>sentinel-datasource-nacos</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    <dependency>
        <groupId>cn.pandora</groupId>
        <artifactId>pandora-common</artifactId>
    </dependency>
</dependencies>
```

**关键约束：**

- **远程调用**：推荐使用 `spring-cloud-starter-openfeign`（与 LoadBalancer、Sentinel 配合，Spring Cloud Alibaba 原生支持），也可按需选 RestClient + Nacos NamingService 或 Dubbo。
- 所有业务微服务通过依赖本模块即可接入 Nacos + Sentinel。
- 提供统一的 `BlockException` 处理，拦截返回 HTTP 429 JSON 响应。

## 3. pandora-domain（领域层）

**职责：** 封装业务领域模型、聚合根、值对象、领域事件、仓储接口。

**示例包结构（User 聚合）：**

```
cn.pandora.domain
├── shared
│   ├── AggregateRoot       聚合根基类
│   ├── Entity              实体基类
│   ├── ValueObject         值对象标记接口
│   ├── DomainEvent         领域事件基类
│   ├── DomainEventPublisher 领域事件发布器接口
│   └── Repository<T, ID>    通用仓储接口
└── user
    ├── User                用户聚合根
    ├── UserId              用户 ID 值对象
    ├── Email               邮箱值对象
    ├── UserStatus          用户状态枚举
    ├── UserRepository      用户仓储接口
    └── event
        ├── UserCreatedEvent
        └── UserStatusChangedEvent
```

**依赖：** 仅依赖 `pandora-common` + JDK，**严禁依赖任何框架**（包括 Spring、MyBatis、Redisson）。

---

## 4. pandora-application（应用层）

**职责：** 编排领域逻辑，提供用例（Use Case）级别的服务。

**示例包结构：**

```
cn.pandora.application
├── user
│   ├── UserApplicationService
│   ├── command
│   │   ├── CreateUserCommand
│   │   └── UpdateUserStatusCommand
│   ├── dto
│   │   └── UserDTO
│   └── assembler
│       └── UserAssembler (MapStruct)
├── file
│   ├── FileApplicationService
│   └── FileService        基础设施抽象接口（由 Infra 实现）
└── ai
    ├── AiApplicationService
    └── AiService          基础设施抽象接口（由 Infra 实现）
```

**依赖：** `pandora-domain` + `pandora-common`；可引入 Spring 的 `@Service`、`@Transactional`、`MapStruct`，但不直接持有任何基础设施实现。

---

## 5. pandora-infrastructure（基础设施层）

**职责：** 实现领域层和应用层定义的所有技术接口。

**核心内容：**

```
cn.pandora.infrastructure
├── persistence             持久化
│   ├── mybatisflex         MyBatis-Flex 配置 + 拦截器
│   │   ├── config/MybatisFlexConfig
│   │   ├── listener/{InsertListener, UpdateListener}   自动填充
│   │   ├── interceptor/DataPermissionInterceptor       数据权限
│   │   └── mapper/*                                     BaseMapper 实现
│   ├── dynamic             dynamic-datasource 配置（@DS 切换）
│   ├── jpa                 Spring Data JPA 仓储（与 MyBatis-Flex 共存）
│   └── repository          领域仓储实现（继承 BaseMapper 并实现 Repository）
├── security                安全
│   ├── config/ResourceServerConfig  多 SecurityFilterChain
│   ├── jwt/JwtUtils                  JJWT 0.12.6 封装
│   ├── filter/LoginUserFilter        jti/tgen 校验 + LoginUser 构建
│   └── aop/PermissionAspect          @RequiresPermission / @RequiresUserType
├── cache                   Redisson 缓存配置
│   ├── config/RedissonConfig         RedissonClient + 集群/单机
│   └── service/CacheService
├── lock                    分布式锁
│   └── DistributedLockService  基于 RedissonClient
├── ratelimit               限流
│   └── RateLimitService        RRateLimiter
├── mq                      消息
│   └── kafka/KafkaConfig + KafkaProducerService
├── mongodb                 文档库
│   └── config/MongoConfig
├── oss                     MinIO
│   └── MinioFileServiceImpl   → 实现应用层 FileService
├── ai                      Spring AI
│   └── OpenAiAiServiceImpl    → 实现应用层 AiService
└── event                   领域事件发布
    └── SpringDomainEventPublisher → 使用 ApplicationEventPublisher
```

**关键实现变化（相对旧版）：**

| 项目 | 旧（MyBatis-Plus） | 新（MyBatis-Flex 1.11.6） |
|------|-----|-----|
| Starter | `mybatis-plus-spring-boot3-starter` | `mybatis-flex-spring-boot4-starter` |
| 自动填充 | `MetaObjectHandler` | `InsertListener` + `UpdateListener` |
| 分页 | `PaginationInnerInterceptor` | 内置 `Page.of(...)` + `mapper.paginate(...)` |
| 数据权限 | `MybatisPlusInterceptor + InnerInterceptor` | 标准 `org.apache.ibatis.plugin.Interceptor` |
| 多数据源 | `dynamic-datasource-spring-boot3-starter` | `dynamic-datasource-spring-boot4-starter 4.5.0` |
| Redis 客户端 | Lettuce（默认）或 Jedis | **只允许 Redisson 4.3.1**（`redisson-spring-boot-starter` + `redisson-spring-data-40`） |

## 6. pandora-api（接口层）

**职责：** 对外暴露 REST API，统一异常处理，文档生成。

**示例包结构：**

```
cn.pandora.api
├── controller
│   ├── UserController
│   ├── FileController
│   ├── AiController
│   └── SysRoleController / SysUserController / ...
├── handler/GlobalExceptionHandler   全局异常处理
└── config/OpenApiConfig             SpringDoc 2.7.0（OpenAPI 3.1）
```

**依赖：** `pandora-application` + `pandora-common`；不直接依赖基础设施。

**规范：**

- Controller 方法只做参数校验 + 调用应用服务 + 返回 `Result`
- 异常由 `GlobalExceptionHandler` 统一转换为 `Result.error(...)`

---

## 7. pandora-auth（OAuth2 授权服务器）

**职责：** 独立的认证授权服务，同时服务 B 端（管理后台）和 C 端（会员/消费者）。

**核心依赖：**

- `spring-security-oauth2-authorization-server` 7.0.4
- `spring-boot-starter-oauth2-authorization-server`
- `justauth` 1.16.7（社交登录）
- `jjwt-api/impl/jackson` 0.12.6（JWT 签发）
- Redisson 4.3.1（白名单/黑名单/代次）
- MyBatis-Flex 1.11.6（管理员/会员数据访问）

**关键子模块：**

```
cn.pandora.auth
├── authentication          认证
│   ├── AuthenticationService   策略编排入口
│   ├── provider
│   │   ├── UsernamePasswordAuthProvider
│   │   ├── SmsCodeAuthProvider
│   │   ├── WechatAuthProvider
│   │   ├── WechatWorkAuthProvider
│   │   ├── QqAuthProvider
│   │   └── WechatQrAuthProvider
│   └── dto/{LoginRequest, LoginResponse}
├── token                   Token 管理
│   ├── AuthTokenService        generateBToken / generateCToken / refresh / logout
│   ├── TokenBlacklistService   jti 黑名单
│   ├── TokenWhitelistService   RefreshToken 白名单
│   └── TgenService             代次控制（强制踢人）
├── user                    用户管理
│   ├── AdminService            B 端账号
│   ├── MemberService           C 端会员
│   └── SocialBindService       社交账号绑定
├── config/SecurityConfig       多 SecurityFilterChain
└── AuthServerApplication       启动类（端口 9100）
```

**端口 / 路径：** 业务 **9100**，运维 **19100**；`/auth/admin/**`（B 端）、`/auth/app/**`（C 端）、`/auth/oauth2/**`（OAuth2 授权码流）。

**Nacos：** 通过 `NACOS_ENABLED` 环境变量控制是否注册，服务名 `pandora-auth-server`。

## 8. pandora-start（单体启动模块）

**职责：** 单体部署入口，聚合 DDD 四层 + 所有基础设施，但**不依赖 Nacos**。

**依赖：**

```xml
<dependencies>
    <dependency>
        <groupId>cn.pandora</groupId>
        <artifactId>pandora-api</artifactId>
    </dependency>
    <dependency>
        <groupId>cn.pandora</groupId>
        <artifactId>pandora-infrastructure</artifactId>
    </dependency>
</dependencies>
```

**启动类：** `cn.pandora.PandoraApplication`

**端口：** 业务 **8080**，运维 **18080**。

**Maven Profile：** `dev`（默认） / `test` / `prod`，过滤 `@spring.profiles.active@` 占位符。

---

## 9. pandora-gateway（API 网关）

**职责：** 微服务模式下的统一入口，路由分发、鉴权、限流。

**核心依赖：**

```xml
<dependencies>
    <dependency>
        <groupId>org.springframework.cloud</groupId>
        <artifactId>spring-cloud-starter-gateway-server-webflux</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-redis-reactive</artifactId>
    </dependency>
    <dependency>
        <groupId>org.redisson</groupId>
        <artifactId>redisson-spring-boot-starter</artifactId>
    </dependency>
    <dependency>
        <groupId>com.alibaba.cloud</groupId>
        <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
    </dependency>
    <dependency>
        <groupId>com.alibaba.cloud</groupId>
        <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
    </dependency>
    <dependency>
        <groupId>com.alibaba.cloud</groupId>
        <artifactId>spring-cloud-starter-alibaba-sentinel</artifactId>
    </dependency>
    <dependency>
        <groupId>com.alibaba.csp</groupId>
        <artifactId>sentinel-datasource-nacos</artifactId>
    </dependency>
</dependencies>
```

**关键点：**

- artifact 新前缀：`spring-cloud-starter-gateway-server-webflux`（旧 `spring-cloud-starter-gateway` 已拆分）
- 配置属性前缀：`spring.cloud.gateway.server.webflux.*`
- `AuthGlobalFilter`：校验 Bearer Token、查询 Redis 黑名单/代次、白名单放行
- 响应式 Redis + Redisson 反应式客户端（无 Lettuce）

**端口：** 业务 **8888**，运维 **18888**；Nacos 服务名 `pandora-gateway`。

## 10. pandora-service-user（用户微服务）

**职责：** 用户限界上下文的微服务部署单元。

**依赖：**

```xml
<dependencies>
    <dependency>
        <groupId>cn.pandora</groupId>
        <artifactId>pandora-api</artifactId>
    </dependency>
    <dependency>
        <groupId>cn.pandora</groupId>
        <artifactId>pandora-infrastructure</artifactId>
    </dependency>
    <dependency>
        <groupId>cn.pandora</groupId>
        <artifactId>pandora-common-cloud</artifactId>
    </dependency>
</dependencies>
```

**启动类：** `cn.pandora.service.user.UserServiceApplication`

**端口：** 业务 **8081**，运维 **18082**；Nacos 服务名 `pandora-service-user`。

**配置（摘要）：**

```yaml
server:
  port: 8081
management:
  server:
    port: 18082

spring:
  application:
    name: pandora-service-user
  profiles:
    active: dev
  cloud:
    nacos:
      server-addr: ${NACOS_SERVER_ADDR:192.168.50.168:8140,192.168.50.168:8550,192.168.50.168:8960}
      discovery:
        group: ${spring.profiles.active}
      config:
        group: ${spring.profiles.active}
        file-extension: yml
```

## 11. 模块加载顺序（单体模式启动日志示例）

```
1. pandora-common                加载公共基础
2. pandora-domain                加载领域模型
3. pandora-application           加载应用服务
4. pandora-infrastructure        加载持久化、缓存、安全、MinIO 等
5. pandora-api                   加载 REST 端点
6. pandora-start                 组合启动
```

微服务模式下增加：

```
- pandora-common-cloud           加载 Nacos / Sentinel
- pandora-service-user           加载用户业务微服务
- pandora-gateway                加载网关过滤器、路由
```
