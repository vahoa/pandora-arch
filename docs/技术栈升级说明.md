## 二、工程命名规范

### 2.1 GroupId / Package / Module

| 维度 | 规则 |
|------|------|
| `groupId` | `cn.pandora` |
| 根 `artifactId` | `pandora-arch` |
| 子模块 `artifactId` | `pandora-<功能>`（`pandora-common`、`pandora-auth`、`pandora-start` 等） |
| Java 包 | `cn.pandora.<模块>.<子层>`（如 `cn.pandora.infrastructure.security`） |
| 模块版本 | 统一 `1.0.0`，由根 `pom.xml` `dependencyManagement` 管控 |

### 2.2 模块清单（10 个）

```
pandora-arch (父)
├── pandora-common         # 通用：Result / 异常 / 工具 / 基础注解 / 安全上下文
├── pandora-common-cloud   # 微服务通用：Nacos + Sentinel + OpenFeign 辅助配置
├── pandora-domain         # DDD 领域层：聚合根 / 值对象 / 领域事件
├── pandora-application    # DDD 应用层：用例编排 / DTO / Command
├── pandora-infrastructure # DDD 基础设施：MyBatis-Flex / Redisson / Kafka / MinIO / Spring AI / Security
├── pandora-api            # DDD 接口层：Controller / 全局异常 / SpringDoc
├── pandora-auth           # 独立认证服务（Spring Authorization Server 7.0.4）
├── pandora-start          # 单体启动模块（端口 8080 / 18080）
├── pandora-gateway        # 微服务网关（Spring Cloud Gateway Server WebFlux，8888 / 18888）
└── pandora-service        # 微服务聚合父模块
    └── pandora-service-user   # 用户微服务（8081 / 18082）
```

## 三、关键代码变更

### 3.1 MyBatis-Plus → MyBatis-Flex

| 旧 API | 新 API |
|--------|--------|
| `@TableName("t")` | `@Table("t")` |
| `@TableId(type = IdType.AUTO)` | `@Id(keyType = KeyType.Auto)` |
| `@TableField("col")` | `@Column("col")` |
| `@TableField(fill=FieldFill.INSERT)` | 删除；改为 `InsertListener` 自动注入 |
| `@TableLogic` | `@Column(isLogicDelete = true)` |
| `com.baomidou.mybatisplus.core.mapper.BaseMapper` | `com.mybatisflex.core.BaseMapper` |
| `ServiceImpl<M, E>` | `com.mybatisflex.spring.service.impl.ServiceImpl` |
| `baseMapper.xxx()` | `mapper.xxx()`（protected 字段改名） |
| `LambdaQueryWrapper.eq(E::f, v)` | `QueryWrapper.create().eq("f", v)` |
| `MybatisPlusInterceptor + InnerInterceptor` | 标准 MyBatis `Interceptor` |
| `MetaObjectHandler` | `InsertListener` / `UpdateListener`（见 `MybatisFlexConfig`） |

> Tip：MyBatis-Flex 的 `mybatis-flex-processor` 会为每个 DO 生成 `XxxTableDef`，支持类型安全的列引用；当前代码暂未启用此生成器以减少耦合，可按需打开。

### 3.2 新增组件

| 组件 | 位置 | 作用 |
|------|------|------|
| `RedissonConfig` | `cn.pandora.infrastructure.config` | Redisson 客户端与 Spring Data Redis 共享连接 |
| `DistributedLockService` | `cn.pandora.infrastructure.lock` | 分布式锁封装（互斥、读写锁、延时队列） |
| `KafkaConfig` | `cn.pandora.infrastructure.config` | 幂等生产者 + 手动 ack 消费者 |
| `KafkaProducerService` | `cn.pandora.infrastructure.mq` | 异步 / 同步发送封装 |
| `MongoConfig` | `cn.pandora.infrastructure.config` | Mongo 审计开关 |
| `MybatisFlexConfig` | `cn.pandora.infrastructure.config` | Flex 全局配置 + Listener + 数据权限拦截器注册 |

### 3.3 JDK 25 关键特性启用

- **ZGC Generational**：`-XX:+UseZGC -XX:+ZGenerational`，毫秒级 STW
- **Virtual Threads**：Spring Boot 4 默认在 `spring.threads.virtual.enabled=true` 时启用
- **Pattern Matching for switch / record patterns**：代码中大量使用 `switch (x) { case A a -> ...; }`
- **Sealed Classes**：`DataScopeStrategy` 等策略类可密封以强约束

  └───────┬────────┘    └────────┬───────────┘    └────────┬─────────┘
          └──────────────────────┼─────────────────────────┘
                                │
      ┌────────┬────────┬───────┴───────┬────────┬────────┐
      ▼        ▼        ▼               ▼        ▼        ▼
  MySQL 9   Redis 7   Kafka 3.8     MongoDB 5  MinIO    Sentinel

```
┌─────────────────────────────┐

└──────────┬──────────────────┘
           │
  ┌────────┴────────┐
  │ pandora-auth    │  :9100 / :19100
  │ (可选注册 Nacos) │
  └─────────────────┘
```

## 八、升级遗留风险与建议

1. **JDK 25 + Lombok**：必须使用 `lombok 1.18.44+`，否则注解处理器编译失败。根 POM 的构建插件已为 Lombok 打开 `jdk.compiler` 模块（`--add-opens`），消除 `sun.misc.Unsafe` 警告。
2. **Spring Boot 4.x**：`@EntityScan` 等部分类包路径变更；`javax` 已全面退出。Spring Framework 7 移除 `spring-jcl`，父 POM 统一兜底引入 `commons-logging`。
3. **Sentinel 适配器包名变化**：Spring MVC 6/7 下 `BlockExceptionHandler` 位置改动，因此 `SentinelConfig` 改为 `@RestControllerAdvice` 统一捕获 `BlockException` 及子类，返回 HTTP 429 JSON。
4. **MyBatis-Flex QueryWrapper**：当前用字符串列名，后续可打开 `mybatis-flex-processor` 使用 `TableDef` 获得类型安全。
5. **Redis 客户端**：全项目只允许 Redisson，若以前业务直接注入 `LettuceConnectionFactory`，需改为 `RedisConnectionFactory`。
6. **Kafka 未默认启用**：`pandora.kafka.enabled=false`，按需开启以避免本地启动时连不到 Broker。
7. **MongoDB 未默认启用**：`pandora.mongodb.enabled=false`，同上。
8. **生产环境**：Redisson 集群模式必须覆盖 `application.yml` 中的 singleServerConfig，否则会误连单机。
9. **网关 artifactId 变更**：`spring-cloud-starter-gateway` → `spring-cloud-starter-gateway-server-webflux`，属性前缀改为 `spring.cloud.gateway.server.webflux.*`。
10. **服务间调用**：OpenFeign 为首选（Spring Cloud 2025.1.0 的 `spring-cloud-starter-openfeign` + `spring-cloud-starter-loadbalancer` + Sentinel 熔断），同时支持 RestClient + Nacos NamingService、Dubbo；新增微服务请优先评估 OpenFeign 是否能满足需求。
