# 服务部署指南

> 本文档覆盖单体服务和微服务的生产环境部署方案。

---

## 一、构建

### 1.1 打包

```bash
# 全量打包（生成所有模块的 jar）
mvn clean package -DskipTests

# 产物位置：
# pandora-auth/target/pandora-auth-1.0.0.jar     ← Auth 服务
# pandora-start/target/pandora-start-1.0.0.jar    ← 单体服务
# pandora-gateway/target/pandora-gateway-1.0.0.jar ← 网关
# pandora-service/pandora-service-user/target/pandora-service-user-1.0.0.jar ← 用户微服务
```

### 1.2 Docker 镜像（可选）

```dockerfile
# Dockerfile（通用模板）
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

```bash
# 构建镜像
docker build -t pandora-auth:latest -f Dockerfile ./pandora-auth
docker build -t pandora-start:latest -f Dockerfile ./pandora-start
docker build -t pandora-gateway:latest -f Dockerfile ./pandora-gateway
```

## 二、单体部署

### 2.1 架构

```
          ┌─────────────┐     ┌───────────────┐
用户  ───→│  Nginx (80)  │────→│ Start (8080)  │────→ MySQL + Redis + MinIO
          └─────────────┘     └───────────────┘
                                      │
                              ┌───────┴────────┐
                              │ Auth (9100)     │
                              └────────────────┘
```

### 2.2 中间件安装

#### MySQL 8.x

```bash
# Docker
docker run -d --name mysql \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=123456 \
  -v /data/mysql:/var/lib/mysql \
  mysql:8.0 --character-set-server=utf8mb4 --collation-server=utf8mb4_general_ci

# 初始化数据库
docker exec -i mysql mysql -u root -p123456 < docs/sql/schema.sql
docker exec -i mysql mysql -u root -p123456 ddd_platform < docs/sql/schema-auth.sql
docker exec -i mysql mysql -u root -p123456 ddd_platform < docs/sql/schema-rbac.sql
docker exec -i mysql mysql -u root -p123456 ddd_platform < docs/sql/schema-member.sql
```

#### Redis

```bash
docker run -d --name redis \
  -p 6379:6379 \
  -v /data/redis:/data \
  redis:7-alpine --appendonly yes
```

#### MinIO

```bash
docker run -d --name minio \
  -p 9000:9000 -p 9001:9001 \
  -v /data/minio:/data \
  -e "MINIO_ROOT_USER=minioadmin" \
  -e "MINIO_ROOT_PASSWORD=minioadmin" \
  minio/minio server /data --console-address ":9001"
```

启动后访问 http://your-server:9001 创建 `corp-web` 存储桶。

### 2.3 部署服务

```bash
# 创建部署目录
mkdir -p /opt/pandora-arch/{auth,start,logs}

# 上传 jar 文件
cp pandora-auth/target/pandora-auth-*.jar /opt/pandora-arch/auth/app.jar
cp pandora-start/target/pandora-start-*.jar /opt/pandora-arch/start/app.jar
```

#### 启动 Auth 服务

```bash
nohup java -jar /opt/pandora-arch/auth/app.jar \
  --spring.datasource.url="jdbc:mysql://localhost:3306/ddd_platform?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true" \
  --spring.datasource.username=root \
  --spring.datasource.password=123456 \
  --spring.data.redis.host=localhost \
  --spring.data.redis.database=1 \
  --auth.jwt.secret=your-production-secret-key-at-least-256-bits \
  > /opt/pandora-arch/logs/auth.log 2>&1 &

echo $! > /opt/pandora-arch/auth/app.pid
```

#### 启动 Start 服务

```bash
nohup java -jar /opt/pandora-arch/start/app.jar \
  --spring.datasource.dynamic.datasource.master.url="jdbc:mysql://localhost:3306/ddd_platform?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true" \
  --spring.datasource.dynamic.datasource.master.username=root \
  --spring.datasource.dynamic.datasource.master.password=123456 \
  --spring.data.redis.host=localhost \
  --spring.data.redis.database=1 \
  --auth.jwt.secret=your-production-secret-key-at-least-256-bits \
  > /opt/pandora-arch/logs/start.log 2>&1 &

echo $! > /opt/pandora-arch/start/app.pid
```

### 2.4 Nginx 反向代理

```nginx
upstream pandora-start {
    server 127.0.0.1:8080;
}

upstream pandora-auth {
    server 127.0.0.1:9100;
}

server {
    listen 80;
    server_name your-domain.com;

    # API 请求
    location /api/ {
        proxy_pass http://pandora-start;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 认证请求
    location /auth/ {
        proxy_pass http://pandora-auth;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # Swagger UI
    location /swagger-ui/ {
        proxy_pass http://pandora-start;
    }

    location /v3/api-docs/ {
        proxy_pass http://pandora-start;
    }

    # 前端静态文件
    location / {
        root /opt/pandora-arch/web;
        try_files $uri $uri/ /index.html;
    }
}
```

### 2.5 Systemd 服务（可选）

```ini
# /etc/systemd/system/pandora-auth.service
[Unit]
Description=DDD Auth Service
After=network.target mysql.service redis.service

[Service]
Type=simple
User=ddd
WorkingDirectory=/opt/pandora-arch/auth
ExecStart=/usr/bin/java -jar -Xms256m -Xmx512m app.jar
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
systemctl enable pandora-auth
systemctl start pandora-auth
systemctl status pandora-auth
```

## 三、微服务部署

### 3.1 架构

```
                            ┌─────────────────┐
                    ┌──────→│  Auth (9100)     │
                    │       └─────────────────┘
          ┌────────┴──┐     ┌─────────────────┐
用户 ────→│Gateway(8888)│───→│  User Svc       │────→ MySQL
          └────────┬──┘     └─────────────────┘      Redis
                    │       ┌─────────────────┐      MinIO
                    └──────→│  其他微服务...    │
                            └─────────────────┘
                                    │
                            ┌───────┴────────┐
                            │  Nacos (8848)   │
                            │  Sentinel(8858) │
                            └────────────────┘
```

### 3.2 Nacos 部署

```bash
# 生产环境建议使用集群模式
docker run -d --name nacos \
  -p 8848:8848 -p 9848:9848 \
  -e MODE=standalone \
  -e SPRING_DATASOURCE_PLATFORM=mysql \
  -e MYSQL_SERVICE_HOST=localhost \
  -e MYSQL_SERVICE_PORT=3306 \
  -e MYSQL_SERVICE_USER=root \
  -e MYSQL_SERVICE_PASSWORD=123456 \
  -e MYSQL_SERVICE_DB_NAME=nacos_config \
  nacos/nacos-server:v2.3.0
```

### 3.3 在 Nacos 中配置

登录 Nacos 控制台 (http://localhost:8848/nacos)，创建配置：

**配置1：`pandora-auth-server.yml` (Group: DEFAULT_GROUP)**

```yaml
spring:
  datasource:
    url: jdbc:mysql://db-host:3306/ddd_platform?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: your-db-password
  data:
    redis:
      host: redis-host
      port: 6379
      database: 1
      password: your-redis-password

auth:
  jwt:
    secret: your-production-jwt-secret-key-at-least-256-bits-long
    expiration: 7200
    refresh-expiration: 604800
  sms:
    mock-enabled: false
```

**配置2：`pandora-service-user.yml`**

```yaml
spring:
  datasource:
    dynamic:
      primary: master
      datasource:
        master:
          url: jdbc:mysql://db-host:3306/ddd_platform?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
          username: root
          password: your-db-password
        slave:
          url: jdbc:mysql://db-slave-host:3306/ddd_platform?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
          username: readonly
          password: your-slave-password
  data:
    redis:
      host: redis-host
      port: 6379
      database: 1
      password: your-redis-password

auth:
  jwt:
    secret: your-production-jwt-secret-key-at-least-256-bits-long
```

**配置3：`pandora-gateway.yml`**

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: auth-service
          uri: lb://pandora-auth-server
          predicates:
            - Path=/auth/**
        - id: user-service
          uri: lb://pandora-service-user
          predicates:
            - Path=/api/**
      default-filters:
        - DedupeResponseHeader=Access-Control-Allow-Credentials Access-Control-Allow-Origin
```

### 3.4 启动顺序

```bash
# 第1步：中间件
# 确保 MySQL、Redis、MinIO、Nacos 全部启动

# 第2步：Auth 服务
java -jar pandora-auth.jar \
  --spring.cloud.nacos.discovery.server-addr=nacos-host:8848 \
  --spring.cloud.nacos.discovery.enabled=true

# 第3步：业务服务
java -jar pandora-service-user.jar \
  --spring.cloud.nacos.discovery.server-addr=nacos-host:8848

# 第4步：Gateway（最后启动，等待上游服务注册完成）
java -jar pandora-gateway.jar \
  --spring.cloud.nacos.discovery.server-addr=nacos-host:8848
```

### 3.5 Docker Compose（推荐）

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    ports: ["3306:3306"]
    environment:
      MYSQL_ROOT_PASSWORD: 123456
    volumes:
      - mysql-data:/var/lib/mysql
      - ./docs/sql:/docker-entrypoint-initdb.d

  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]
    command: redis-server --appendonly yes

  minio:
    image: minio/minio
    ports: ["9000:9000", "9001:9001"]
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin
    command: server /data --console-address ":9001"

  nacos:
    image: nacos/nacos-server:v2.3.0
    ports: ["8848:8848", "9848:9848"]
    environment:
      MODE: standalone

  pandora-auth:
    build: ./pandora-auth
    ports: ["9100:9100"]
    depends_on: [mysql, redis, nacos]
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/ddd_platform?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
      SPRING_DATA_REDIS_HOST: redis
      SPRING_DATA_REDIS_DATABASE: 1
      NACOS_ENABLED: "true"
      SPRING_CLOUD_NACOS_DISCOVERY_SERVER_ADDR: nacos:8848

  pandora-start:
    build: ./pandora-start
    ports: ["8080:8080"]
    depends_on: [mysql, redis, pandora-auth]
    environment:
      SPRING_DATA_REDIS_HOST: redis
      SPRING_DATA_REDIS_DATABASE: 1

  pandora-gateway:
    build: ./pandora-gateway
    ports: ["8888:8888"]
    depends_on: [nacos, pandora-auth]
    environment:
      SPRING_CLOUD_NACOS_DISCOVERY_SERVER_ADDR: nacos:8848

volumes:
  mysql-data:
```

```bash
docker-compose up -d
```

## 四、生产环境安全清单

| 项目 | 说明 |
|------|------|
| ✅ 修改 JWT Secret | 使用至少 256 位的随机字符串，通过环境变量注入 |
| ✅ 修改数据库密码 | 不要使用默认的 123456 |
| ✅ 修改 Redis 密码 | 设置 requirepass |
| ✅ 修改 MinIO 密码 | 不要使用默认的 minioadmin |
| ✅ 关闭 SMS Mock | 设置 `auth.sms.mock-enabled=false`，配置真实短信服务商 |
| ✅ 关闭 Swagger | 生产环境设置 `springdoc.api-docs.enabled=false` |
| ✅ 配置 HTTPS | Nginx 配置 SSL 证书 |
| ✅ 配置社交登录 | 填入真实的微信/QQ AppID 和 AppSecret |
| ✅ 配置 JVM 参数 | `-Xms512m -Xmx1024m -XX:+UseG1GC` |
| ✅ 配置日志 | 使用 logback-spring.xml 配置文件滚动和日志级别 |
