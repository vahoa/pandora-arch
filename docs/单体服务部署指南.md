# 单体服务部署指南

## 项目背景

本项目基于 **JDK 25 + Spring Boot 4.0.5 + OAuth2 + DDD** 构建，采用 Maven 多模块结构。单体模式通过 `pandora-start` 模块启动，集成 DDD 四层 + 所有基础设施，不依赖 Nacos。

> 基线：JDK 25 · Spring Boot 4.0.5 · Spring Authorization Server 7.0.4 · MyBatis-Flex 1.11.6 · Redisson 4.3.1

---

## 1. 环境要求

| 组件 | 版本要求 | 说明 |
|------|---------|-----|
| JDK | **25（LTS）** | 虚拟线程默认 + ZGC Generational |
| Maven | 3.9+ | — |
| MySQL | 8.x / 9.x | 驱动 `mysql-connector-j 9.6.0` |
| Redis | 7.x | 仅 Redisson 客户端（禁用 Lettuce/Jedis） |
| MinIO | 最新版 | 可选（对象存储），端点 127.0.0.1:9000，桶 `corp-web` |
| MongoDB | 5.x/6.x/7.x | 可选（`pandora.mongodb.enabled=true`） |
| Kafka | 3.8.x | 可选（`pandora.kafka.enabled=true`，KRaft 模式） |

单体模式**不需要** Nacos。

## 4. 服务配置说明

### 4.1 主应用配置（pandora-start）

配置文件：`pandora-start/src/main/resources/application.yml`

| 配置项 | 说明 |
|--------|-----|
| `server.port` | 8080 |
| `management.server.port` | 18080 |
| `spring.datasource.dynamic` | 动态数据源（master/slave，基于 `dynamic-datasource-spring-boot4-starter 4.5.0`） |
| `spring.data.redis` | Redis 连接（通过 `redisson-spring-data-40` 使用 Redisson） |
| `mybatis-flex.*` | MyBatis-Flex 配置 |
| `pandora.minio.*` | MinIO 对象存储 |
| `pandora.kafka.enabled` | Kafka 开关（默认 false） |
| `pandora.mongodb.enabled` | MongoDB 开关（默认 false） |
| `spring.security.oauth2.resourceserver.jwt.*` | OAuth2 资源服务器配置（JWT HMAC） |

### 4.2 认证服务配置（pandora-auth）

配置文件：`pandora-auth/src/main/resources/application.yml`

| 配置项 | 说明 |
|--------|-----|
| `server.port` | 9100 |
| `management.server.port` | 19100 |
| `spring.application.name` | pandora-auth-server |
| `auth.jwt.*` | JWT 签名密钥、过期时间 |
| `auth.channel.*` | 多渠道认证开关 |
| `justauth.*` | 社交登录平台配置（微信 / QQ / 企业微信） |
| `NACOS_ENABLED` | 是否注册 Nacos（单体模式建议 false） |

## 6. 启动步骤（顺序）

### 6.1 前置准备

1. 启动 MySQL、Redis，以及可选的 MinIO / MongoDB / Kafka
2. 执行数据库初始化脚本（见第 3 节）

### 6.2 方式一：Maven 启动

```bash
# 1. 启动认证服务器（端口 9100）
cd pandora-auth && mvn spring-boot:run

# 2. 启动单体主应用（端口 8080）
cd pandora-start && mvn spring-boot:run
```

### 6.3 方式二：JAR 启动

```bash
# 建议加上虚拟线程 + ZGC 分代 + 打开 Lombok 需要的模块
JAVA_OPTS="-XX:+UseZGC -XX:+ZGenerational \
  --add-opens=jdk.compiler/com.sun.tools.javac.processing=ALL-UNNAMED"

java $JAVA_OPTS -jar pandora-auth/target/pandora-auth-1.0.0.jar
java $JAVA_OPTS -jar pandora-start/target/pandora-start-1.0.0.jar
```

> **顺序说明**：必须先启动 `pandora-auth`（9100），再启动 `pandora-start`（8080），因为资源服务器需要通过 auth 服务的 HMAC 密钥进行 JWT 验签（一般通过共享配置）。

## 8. 认证接口测试（curl 示例）

### 8.1 B 端用户名密码登录


## 9. 多数据源 @DS 用法

```java
@Service
public class OrderService {
    @DS("master")
    public void createOrder(Order o) { ... }

    @DS("slave")
    public List<Order> listForRead() { ... }
}
```

配置：

```yaml
spring:
  datasource:
    dynamic:
      primary: master
      strict: false
      datasource:
        master:
          url: jdbc:mysql://localhost:3306/ddd_platform?...
          username: root
          password: 123456
          driver-class-name: com.mysql.cj.jdbc.Driver
        slave:
          url: jdbc:mysql://localhost:3306/ddd_platform?...
          username: root
          password: 123456
          driver-class-name: com.mysql.cj.jdbc.Driver
```

> Starter：**`dynamic-datasource-spring-boot4-starter:4.5.0`**。

## 11. 生产环境部署注意事项

### 11.1 安全配置

- **修改 JWT Secret**：`AUTH_JWT_SECRET` 必须设置为强随机字符串（≥ 32 字节），切勿使用默认值
- **关闭短信模拟模式**：`SMS_MOCK_ENABLED=false`，接入真实短信网关
- **配置正式的社交登录凭据**：生产环境需配置微信、QQ、企业微信等平台的正式 Client ID / Secret
- **开启 HTTPS**：Nginx 配置 SSL 证书，业务端口不对公网暴露

### 11.2 JVM 调优

推荐启动参数（JDK 25）：

```bash
-Xms1g -Xmx2g \
-XX:+UseZGC -XX:+ZGenerational \
-XX:MaxDirectMemorySize=256m \
-Dspring.threads.virtual.enabled=true \
-Dfile.encoding=UTF-8
```

### 11.3 Nginx 反向代理

```nginx
upstream pandora_start {
    server 127.0.0.1:8080;
}

upstream pandora_auth {
    server 127.0.0.1:9100;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate     /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;

    location /api/ {
        proxy_pass http://pandora_start;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /auth/ {
        proxy_pass http://pandora_auth;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```
