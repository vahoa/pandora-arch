# 单体服务部署指南

## 项目背景

本项目是一个基于 **Spring Boot 3.2.3 + OAuth2 + DDD** 的架构底座，采用 Maven 多模块结构。单体模式通过 `pandora-start` 模块启动，集成所有功能。

---

## 1. 环境要求

| 组件 | 版本要求 |
|------|----------|
| JDK | 17+ |
| Maven | 3.8+ |
| MySQL | 8.x（localhost:3306，root/123456）|
| Redis | 6.x+（localhost:6379）|
| MinIO | 127.0.0.1:9000，minioadmin/minioadmin，bucket: corp-web |
| OpenAI API Key | 可选，用于 Spring AI 功能 |

---

## 2. 数据库初始化

执行以下命令初始化数据库：

```bash
mysql -u root -p123456 < docs/sql/schema.sql
mysql -u root -p123456 < docs/sql/schema-auth.sql
```

> 请确保 MySQL 服务已启动，且 `docs/sql/` 目录下存在上述 SQL 脚本文件。

## 3. 服务配置说明

### 3.1 主应用配置（pandora-start）

配置文件路径：`pandora-start/src/main/resources/application.yml`

| 配置项 | 说明 |
|--------|------|
| `spring.datasource.dynamic` | 动态数据源（master/slave）|
| `spring.data.redis` | Redis 连接配置 |
| `spring.jpa` | JPA 配置（ddl-auto: none）|
| `spring.ai.openai` | Spring AI 配置（可选，通过环境变量配置）|
| `minio.*` | MinIO 对象存储配置 |
| `mybatis-plus.*` | MyBatis-Plus 配置 |
| `spring.security.oauth2.resourceserver.jwt.issuer-uri` | OAuth2 资源服务器 JWT 签发地址：http://localhost:9100 |

### 3.2 认证服务配置（pandora-auth）

配置文件路径：`pandora-auth/src/main/resources/application.yml`

| 配置项 | 说明 |
|--------|------|
| 端口 | 9100 |
| `auth.*` | 多渠道认证配置 |
| 社交登录 | 通过环境变量配置各平台 Client ID/Secret |
| 短信模拟模式 | SMS_MOCK_ENABLED=true |
| JWT | JWT 签名、过期时间等配置 |

## 4. 编译构建

在项目根目录执行：

```bash
mvn clean package -DskipTests
```

构建完成后，可执行 JAR 位于：
- `pandora-auth/target/pandora-auth-1.0.0.jar`
- `pandora-start/target/pandora-start-1.0.0.jar`

---

## 5. 启动步骤（顺序）

### 5.1 前置准备

1. 启动 MySQL、Redis、MinIO，确认服务可用
2. 执行数据库初始化脚本（见第 2 节）

### 5.2 方式一：Maven 启动

```bash
# 1. 启动认证服务器（端口 9100）
cd pandora-auth && mvn spring-boot:run

# 2. 新开终端，启动主应用（端口 8080）
cd pandora-start && mvn spring-boot:run
```

### 5.3 方式二：JAR 启动

```bash
# 1. 启动认证服务器
java -jar pandora-auth/target/pandora-auth-1.0.0.jar

# 2. 启动主应用
java -jar pandora-start/target/pandora-start-1.0.0.jar
```

> **注意**：必须先启动认证服务器（9100），再启动主应用（8080），因为主应用依赖 OAuth2 资源服务器的 JWT 校验。

## 7. 认证接口测试（curl 示例）

### 7.1 用户名密码登录

```bash
curl -X POST "http://localhost:9100/auth/oauth/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=password&username=admin&password=123456&client_id=web&client_secret=secret"
```

### 7.2 发送短信验证码

```bash
curl -X POST "http://localhost:9100/auth/sms/send" \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800138000"}'
```

### 7.3 短信验证码登录

```bash
curl -X POST "http://localhost:9100/auth/oauth/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=sms&phone=13800138000&code=123456&client_id=web&client_secret=secret"
```

### 7.4 社交登录 URL 获取

```bash
# 微信登录
curl "http://localhost:9100/auth/oauth2/authorization/wechat"

# QQ 登录
curl "http://localhost:9100/auth/oauth2/authorization/qq"
```

### 7.5 Token 刷新

```bash
curl -X POST "http://localhost:9100/auth/oauth/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=refresh_token&refresh_token=YOUR_REFRESH_TOKEN&client_id=web&client_secret=secret"
```

### 7.6 OAuth2 客户端凭证获取令牌

```bash
curl -X POST "http://localhost:9100/auth/oauth/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials&client_id=web&client_secret=secret"
```

## 8. 多数据源配置说明

### 8.1 @DS 注解使用

使用 MyBatis-Plus 动态数据源时，通过 `@DS` 注解指定数据源：

```java
@Service
@DS("master")  // 主库
public class UserServiceImpl implements UserService {
    // ...
}

@Service
@DS("slave")   // 从库（读库）
public class OrderQueryServiceImpl implements OrderQueryService {
    // ...
}
```

- `master`：主库，用于写操作
- `slave`：从库，用于读操作

### 8.2 新增数据源步骤

1. 在 `application.yml` 的 `spring.datasource.dynamic.datasource` 下新增数据源配置
2. 在 `spring.datasource.dynamic.druid` 中配置连接池参数（如使用 Druid）
3. 在业务类上使用 `@DS("新数据源名称")` 指定数据源

## 10. 生产环境部署注意事项

### 10.1 安全配置

- **修改 JWT Secret**：必须将 `AUTH_JWT_SECRET` 设置为强随机字符串，切勿使用默认值
- **关闭短信模拟模式**：设置 `SMS_MOCK_ENABLED=false`，接入真实短信服务商
- **配置正式的社交登录凭据**：生产环境需配置微信、QQ 等平台的正式 Client ID 和 Secret

### 10.2 数据库连接池调优

根据实际并发量调整 `spring.datasource.dynamic.druid` 中的参数：

- `initial-size`：初始连接数
- `min-idle`：最小空闲连接数
- `max-active`：最大活跃连接数
- `max-wait`：获取连接最大等待时间

### 10.3 Nginx 反向代理配置示例

```nginx
# 主应用
upstream ddd_start {
    server 127.0.0.1:8080;
}

# 认证服务
upstream ddd_auth {
    server 127.0.0.1:9100;
}

server {
    listen 80;
    server_name your-domain.com;

    location /api/ {
        proxy_pass http://ddd_start/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /auth/ {
        proxy_pass http://ddd_auth/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

> 生产环境建议使用 HTTPS，并配置 SSL 证书。
